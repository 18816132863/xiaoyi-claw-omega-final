## 三、统一入口接口

### 3.1 策略引擎.评估策略()

**功能**：所有请求必须经过此接口

**调用链**：
```
1. 加载指标 () → 读取聚合指标
2. 风险分类器.分类() → 返回风险评估
3. 权限引擎.检查权限() → 返回权限检查结果
4. Token预算管理器.获取决策预算() → 返回剩余预算
5. 成本预算管理器.获取决策预算() → 返回成本预算
6. 高风险守卫.守卫() → 返回守卫决策
7. 根据指标调整() → 基于成功率/失败率调整决策
```

**返回决策对象**：
```python
{
    "decision": "allow",  # 允许 / 拒绝 / 降级 / 审核
    "allowed": True,
    "risk_level": "low",
    "risk_score": 0.0,
    "token_budget": 16000,
    "cost_budget": 50.0,
    "allowed_capabilities": [...],
    "blocked_capabilities": [...],
    "requires_review": False,
    "reason": "...",
    "mitigations": [...],
    "metrics": {
        "task_success_rate": 0.70,
        "skill_failure_rate": 0.20,
        "memory_hit_rate": 0.75,
        "overall_health": "good"
    }
}
```

### 3.2 技能路由器.选择技能()

**功能**：技能选择必须经过此接口

**调用链**：
```
1. 检查并重新加载指标() → 检查文件修改时间
2. 注册表.重新加载() → 获取最新状态
3. 按意图搜索候选技能
4. 按Profile过滤分类
5. 按状态过滤（排除禁用/弃用）
6. 按标签过滤
7. 按治理决策过滤
8. 按指标排序() → 低失败率优先
```

**返回选择结果**：
```python
{
    "success": True,
    "skill_id": "search",
    "confidence": 0.9,
    "reason": "按意图匹配",
    "profile": "developer",
    "category": "search",
    "status": "stable",
    "tags": [...]
}
```

### 3.3 工作流引擎.运行工作流()

**功能**：工作流执行必须经过此接口

**注入组件**：
- 回退策略
- 回滚管理器
- 检查点存储

**执行流程**：
```
1. 构建DAG() → 构建依赖图
2. 加载检查点（如果恢复）
3. 执行DAG():
   - 步骤前保存检查点
   - 执行步骤():
     - 执行动作() → 保留原始错误语义
     - 失败时回退策略.决定()
   - 步骤后保存检查点
4. 保存最终检查点
```

**返回结果**：
```python
WorkflowResult:
    - workflow_id: 工作流ID
    - status: 完成状态
    - step_results: 步骤结果
    - failed_step_id: 失败步骤ID
    - reason: 原因
    - retry_count: 重试次数
    - fallback_used: 是否使用回退
    - rollback_used: 是否使用回滚
    - checkpoint_id: 检查点ID
```

```
    ↓
弃用管理器.弃用()
    ↓
技能路由器.选择技能(意图, 配置)
    ├── 注册表.重新加载() 获取最新状态
    ├── 过滤禁用/弃用技能
    ├── 接入预算策略检查
    └── 按指标排序候选技能
```

### 4.2 工作流恢复链主链

```
工作流引擎.运行工作流()
    ├── 注入回退策略、回滚管理器、检查点存储
    │
    ├── 执行DAG():
    │   ├── 步骤前保存检查点
    │   ├── 执行步骤():
    │   │   ├── 执行动作() → 保留原始错误语义
    │   │   └── 失败时回退策略.决定()
    │   │       ├── 重试
    │   │       ├── 回退 → 使用回退技能
    │   │       ├── 跳过
    │   │       └── 中止
    │   └── 步骤后保存检查点
    │
    └── 工作流结果:
        ├── 失败步骤ID
        ├── 原因
        ├── 重试次数
        ├── 是否使用回退
        ├── 是否使用回滚
        └── 检查点ID
```

```
生成指标.py

    └── 评估聚合器 → aggregated_metrics.json

策略引擎.评估策略()
    ├── 加载指标() 加载聚合指标
    ├── 根据指标调整() 基于指标调整决策
    │   ├── 任务成功率 < 50% → 降级
    │   ├── 技能失败率 > 30% → 降级
    │   └── 健康状态 == 差 → 降级
    └── 返回决策对象

技能路由器.__初始化__()
    └── 自动加载指标() 自动加载指标

技能路由器.选择技能()
    ├── 检查指标重新加载() 检查修改时间重新加载
    └── 按指标排序() 按指标排序
        ├── 失败率升序（低失败率优先）
```
