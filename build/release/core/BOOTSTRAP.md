# BOOTSTRAP.md - 终极鸽子王会话启动引导

**版本: V26.0**

## 目的
定义每次会话启动时的初始化流程，确保系统状态一致、记忆完整加载。

## 适用范围
所有新会话启动、会话重置、系统重启。

## 启动流程

```
会话启动
    ↓
┌─────────────────────────────────────┐
│ Phase 1: 核心身份加载               │
│ - 读取 IDENTITY.md                  │
│ - 读取 SOUL.md                      │
│ - 确认版本与模式                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Phase 2: 用户上下文加载             │
│ - 读取 USER.md                      │
│ - 读取 MEMORY.md                    │
│ - 加载记忆索引                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Phase 3: 系统状态检查               │
│ - 读取 HEARTBEAT.md                 │
│ - 检查健康度状态                    │
│ - 确认自动升级状态                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Phase 4: 能力初始化                 │
│ - 加载运行时配置                    │
│ - 加载安全边界                      │
│ - 加载技能路由                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ Phase 5: 问候与就绪                 │
│ - 生成个性化问候                    │
│ - 报告系统状态（可选）              │
│ - 加载交互引导                      │
│ - 等待用户指令                      │
└─────────────────────────────────────┘
    ↓
会话就绪
```

## 启动检查清单

### 必读文件（按顺序）
| 顺序 | 文件 | 用途 | 失败处理 |
|------|------|------|----------|
| 1 | IDENTITY.md | 身份确认 | 使用默认身份 |
| 2 | SOUL.md | 角色加载 | 使用默认角色 |
| 3 | USER.md | 用户画像 | 创建新画像 |
| 4 | MEMORY.md | 记忆索引 | 初始化记忆 |
| 5 | HEARTBEAT.md | 系统状态 | 执行健康检查 |
| 6 | INTERACTION_GUIDE.md | 交互引导 | 使用默认引导 |

### 可选文件
| 文件 | 用途 | 加载条件 |
|------|------|----------|
| ARCHITECTURE_V8.md | 架构参考 | 需要架构信息时 |
| SYSTEM_CAPABILITY_MATRIX_V8.json | 能力矩阵 | 需要能力详情时 |
| UPGRADE_SUMMARY_V8.md | 升级历史 | 需要版本信息时 |

## 启动模式

### 标准启动
```
完整加载 → 健康检查 → 就绪
```
适用于：正常会话、日常使用

### 快速启动
```
核心加载 → 跳过检查 → 就绪
```
适用于：紧急响应、快速任务

### 维护启动
```
完整加载 → 深度检查 → 报告生成 → 就绪
```
适用于：系统维护、问题诊断

## 启动超时

| 阶段 | 超时时间 | 超时处理 |
|------|----------|----------|
| Phase 1 | 5s | 使用缓存身份 |
| Phase 2 | 10s | 使用基础画像 |
| Phase 3 | 15s | 跳过检查继续 |
| Phase 4 | 10s | 使用默认配置 |
| Phase 5 | 5s | 简化问候 |

## 启动失败恢复

| 失败类型 | 恢复策略 |
|----------|----------|
| 核心文件缺失 | 使用内置默认值 |
| 记忆加载失败 | 初始化新记忆系统 |
| 健康检查失败 | 标记待修复，继续启动 |
| 配置加载失败 | 使用安全默认配置 |

## 启动日志

每次启动记录以下信息：
- 启动时间（UTC）
- 启动模式
- 加载文件列表
- 健康检查结果
- 耗时统计

日志路径：`~/.openclaw/logs/bootstrap.log`
