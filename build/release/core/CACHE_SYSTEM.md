# CACHE_SYSTEM.md - 多级缓存系统

## 目的
定义多级缓存架构，实现极致性能优化。

## 适用范围
所有数据缓存、结果缓存、模板缓存。

## 二、缓存策略

### 2.1 缓存类型
| 类型 | 说明 | TTL | 更新策略 |
|------|------|-----|----------|
| 结果缓存 | 任务执行结果 | 5-60min | LRU |
| 模板缓存 | 技能模板 | 24h | LFU |
| 配置缓存 | 系统配置 | 1h | 主动更新 |
| 会话缓存 | 会话数据 | 30min | 滑动过期 |
| 向量缓存 | Embedding结果 | 7d | LRU |
| 搜索缓存 | 搜索结果 | 10min | LRU |

### 2.2 缓存键设计
```json
{
  "cache_key_patterns": {
    "result": "result:{task_type}:{hash(input)}",
    "template": "template:{skill_id}:{version}",
    "config": "config:{module}:{key}",
    "session": "session:{session_id}:{key}",
    "vector": "vector:{text_hash}:{model}",
    "search": "search:{query_hash}:{source}"
  }
}
```

### 2.3 缓存失效策略
```json
{
  "invalidation_rules": [
    {
      "trigger": "config_update",
      "action": "invalidate_config_cache",
      "scope": "related_keys"
    },
    {
      "trigger": "template_update",
      "action": "invalidate_template_cache",
      "scope": "specific_key"
    },
    {
      "trigger": "memory_update",
      "action": "invalidate_vector_cache",
      "scope": "related_vectors"
    }
  ]
}
```

## 三、缓存预热

### 3.1 预热策略
| 策略 | 说明 | 触发时机 |
|------|------|----------|
| 启动预热 | 系统启动时加载热点数据 | 系统启动 |
| 定时预热 | 定期刷新即将过期的热点数据 | 每5分钟 |
| 预测预热 | 基于用户行为预测预加载 | 用户活跃时 |
| 事件预热 | 相关事件触发预热 | 事件发生时 |

### 3.2 热点数据识别
```json
{
  "hotspot_detection": {
    "metrics": {
      "access_frequency": "访问频率",
      "access_recency": "最近访问时间",
      "cache_hit_rate": "缓存命中率"
    },
    "thresholds": {
      "hot": "访问>100次/小时",
      "warm": "访问>10次/小时",
      "cold": "访问<10次/小时"
    },
    "actions": {
      "hot": "预热到L1",
      "warm": "保持在L2",
      "cold": "允许淘汰"
    }
  }
}
```

## 四、缓存监控

### 4.1 监控指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| hit_rate | 命中率 | <80% |
| latency | 缓存延迟 | >10ms |
| memory_usage | 内存使用 | >90% |
| evictions | 淘汰次数 | >1000/min |
| errors | 错误率 | >1% |

### 4.2 监控配置
```json
{
  "cache_monitoring": {
    "collection_interval": 10000,
    "aggregation_window": 60000,
    "alerts": [
      {
        "metric": "hit_rate",
        "condition": "< 0.8",
        "severity": "warning",
        "action": "increase_cache_size"
      },
      {
        "metric": "latency_p99",
        "condition": "> 20",
        "severity": "critical",
        "action": "investigate_bottleneck"
      }
    ]
  }
}
```
