# INTERACTION_GUIDE.md - 交互引导模块

## 目的
定义自然交互规范，提供最佳交互体验。

## 适用范围
所有用户交互场景。

---

## 一、交互原则

### 1. 自然优先
- 支持自然语言输入
- 无需记忆命令
- 智能意图理解

### 2. 渐进引导
- 新手友好
- 逐步深入
- 按需展示

### 3. 即时反馈
- 实时响应
- 进度可见
- 结果明确

### 4. 容错设计
- 错误友好提示
- 自动纠正建议
- 历史可追溯

---

## 二、交互模式

### 自然语言模式
```
用户: 帮我分析一下最近的销售数据
系统: [自动理解意图，执行分析，返回结果]
```

### 命令模式
```
用户: /analyze --data sales --period monthly
系统: [执行命令，返回结果]
```

### 对话模式
```
用户: 我想做一个项目
系统: 好的，请告诉我项目的类型和目标
用户: 是一个电商网站
系统: [继续对话，逐步明确需求]
```

---

## 三、引导策略

### 新用户引导
1. 欢迎介绍
2. 能力展示
3. 首次任务引导
4. 模式选择

### 任务引导
1. 任务理解确认
2. 执行方案展示
3. 进度实时反馈
4. 结果呈现与确认

### 错误引导
1. 友好错误提示
2. 可能原因分析
3. 解决方案建议
4. 重试或替代方案

---

## 四、响应规范

### 响应格式
```
[状态] 主要内容

详细信息（可选）

💡 建议（可选）
```

### 响应时间
| 场景 | 目标时间 |
|------|----------|
| 简单查询 | <100ms |
| 一般任务 | <1s |
| 复杂分析 | <10s |
| 长任务 | 进度反馈 |

### 响应类型
| 类型 | 使用场景 |
|------|----------|
| 直接回答 | 事实性问题 |
| 结构化输出 | 数据分析 |
| 步骤引导 | 复杂任务 |
| 对话式 | 需求澄清 |
