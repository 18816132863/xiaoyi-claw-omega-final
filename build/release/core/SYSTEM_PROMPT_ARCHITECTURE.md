# SYSTEM_PROMPT_ARCHITECTURE.md - 系统提示架构

## 目的
定义系统提示架构分层，确保提示结构清晰，便于维护和排错。

## 适用范围
所有系统提示的构建和管理。

## 提示分层架构

### 第一层：核心身份层
| 特征 | 说明 |
|------|------|
| 不可覆盖 | 任何情况下不可覆盖 |
| 永久存在 | 始终存在于提示中 |
| 定义身份 | 定义系统基本身份 |

**内容**:
- 系统名称和版本
- 基本角色定义
- 核心能力边界

### 第二层：行为规则层
| 特征 | 说明 |
|------|------|
| 不可覆盖 | 安全规则不可覆盖 |
| 按需加载 | 根据任务类型加载 |
| 定义行为 | 定义系统行为规范 |

**内容**:
- 安全边界规则
- 执行策略规则
- 风险处理规则

### 第三层：任务策略层
| 特征 | 说明 |
|------|------|
| 可动态注入 | 根据任务动态注入 |
| 任务相关 | 与具体任务相关 |
| 定义策略 | 定义任务执行策略 |

**内容**:
- 任务类型策略
- 技能路由策略
- 输出格式策略

### 第四层：临时场景层
| 特征 | 说明 |
|------|------|
| 完全动态 | 完全动态生成 |
| 场景相关 | 与具体场景相关 |
| 定义上下文 | 定义当前上下文 |

**内容**:
- 用户当前请求
- 会话上下文
- 检索结果
- 工具返回

## 层级注入规则

### 核心身份层注入
| 规则 | 说明 |
|------|------|
| 始终注入 | 任何情况下都注入 |
| 位置固定 | 始终在最前面 |
| 不可修改 | 不允许动态修改 |

### 行为规则层注入
| 规则 | 说明 |
|------|------|
| 按需加载 | 根据任务类型加载 |
| 安全优先 | 安全规则优先加载 |
| 可扩展 | 可添加新规则 |

### 任务策略层注入
| 规则 | 说明 |
|------|------|
| 动态生成 | 根据任务动态生成 |
| 可覆盖 | 可被用户要求覆盖 |
| 可扩展 | 可添加新策略 |

### 临时场景层注入
| 规则 | 说明 |
|------|------|
| 完全动态 | 完全动态生成 |
| 可裁剪 | 超长时可裁剪 |
| 可压缩 | 可压缩处理 |

## 提示组装格式

```
=== [LAYER 1: IDENTITY] ===
{核心身份内容}
=== [END LAYER 1] ===

=== [LAYER 2: RULES] ===
{行为规则内容}
=== [END LAYER 2] ===

=== [LAYER 3: STRATEGY] ===
{任务策略内容}
=== [END LAYER 3] ===

=== [LAYER 4: CONTEXT] ===
{临时场景内容}
=== [END LAYER 4] ===
```

## 层级冲突处理

### 冲突优先级
| 优先级 | 层级 |
|--------|------|
| 1 | 核心身份层 |
| 2 | 行为规则层 |
| 3 | 任务策略层 |
| 4 | 临时场景层 |

### 冲突处理
| 冲突情况 | 处理方式 |
|----------|----------|
| 低层覆盖高层 | 拒绝 |
| 高层覆盖低层 | 允许 |
| 同层冲突 | 按具体规则处理 |

## 提示版本管理

### 版本标识
```json
{
  "promptVersion": "1.0.0",
  "layers": {
    "identity": "1.0.0",
    "rules": "1.1.0",
    "strategy": "1.0.0",
    "context": "dynamic"
  }
}
```

### 版本控制
| 操作 | 说明 |
|------|------|
| 版本记录 | 记录每次变更 |
| 回滚支持 | 支持版本回滚 |
| 差异对比 | 支持差异对比 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 层级缺失 | 使用默认内容 |
| 内容超长 | 按优先级裁剪 |
| 格式错误 | 自动修正 |

## 维护方式
- 新增层级: 添加到层级架构表
- 调整注入规则: 更新注入规则表
- 新增冲突处理: 更新冲突处理表
