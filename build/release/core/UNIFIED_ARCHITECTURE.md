# 统一架构文档 - V7.1.0

## 架构愿景

构建一个具备**思考、恢复、审查、管控、自动化**能力的智能系统架构。

## 六层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    L1: Core 核心层                          │
│  认知系统 | 身份定义 | 规则标准 | 策略文档                    │
├─────────────────────────────────────────────────────────────┤
│                    L2: Memory Context 记忆层                │
│  会话记忆 | 向量存储 | 检索策略 | 索引管理                    │
├─────────────────────────────────────────────────────────────┤
│                    L3: Orchestration 编排层                 │
│  任务引擎 | 工作流引擎 | 优先级队列 | 技能路由                │
├─────────────────────────────────────────────────────────────┤
│                    L4: Execution 执行层                     │
│  技能网关 | 执行器池 | 速率限制 | 循环防护                    │
├─────────────────────────────────────────────────────────────┤
│                    L5: Governance 治理层                    │
│  安全审计 | 权限管理 | 恢复性 | 审查性 | 规则管控            │
├─────────────────────────────────────────────────────────────┤
│                    L6: Infrastructure 基础设施层            │
│  自动化 | 技能注册表 | CLI工具 | 脚本库                      │
└─────────────────────────────────────────────────────────────┘
```

## 核心能力矩阵

### 1. 思考能力 (L1 Core)

| 模块 | 功能 | 文件 |
|------|------|------|
| 推理引擎 | 6种推理模式 | core/cognition/reasoning.py |
| 决策系统 | 3种决策方法 | core/cognition/decision.py |
| 规划引擎 | 任务分解执行 | core/cognition/planning.py |
| 反思系统 | 自我评估改进 | core/cognition/reflection.py |
| 学习系统 | 知识积累迁移 | core/cognition/learning.py |

**推理模式**:
- 演绎推理: 从一般到特殊
- 归纳推理: 从特殊到一般
- 溯因推理: 从结果推断原因
- 类比推理: 基于相似性推断
- 因果推理: 分析因果关系
- 反事实推理: 假设性思考

**决策方法**:
- 加权求和法: 简单快速决策
- AHP层次分析: 多准则决策
- TOPSIS: 复杂多属性决策

### 2. 恢复能力 (L5 Governance)

| 模块 | 功能 | 文件 |
|------|------|------|
| 状态恢复 | 恢复点管理 | governance/recovery/state_recovery.py |
| 故障恢复 | 故障检测处理 | governance/recovery/fault_recovery.py |
| 回滚管理 | 变更回滚 | governance/recovery/rollback_manager.py |

**恢复策略**:
- 自动备份: 变更前自动创建恢复点
- 故障检测: 实时监控系统状态
- 智能恢复: 根据故障类型选择恢复策略
- 回滚支持: 支持单步和批量回滚

### 3. 审查能力 (L5 Governance)

| 模块 | 功能 | 文件 |
|------|------|------|
| 变更审查 | 风险评级 | governance/review/change_review.py |
| 决策审查 | 推理质量评估 | governance/review/decision_review.py |
| 合规审查 | 规则检查 | governance/review/compliance_review.py |

**审查维度**:
- 安全性: 检查是否涉及敏感数据
- 合规性: 检查是否符合规范
- 风险评估: 评估变更风险级别
- 建议生成: 自动生成改进建议

### 4. 规则管控 (L5 Governance)

| 模块 | 功能 | 文件 |
|------|------|------|
| 规则引擎 | 规则执行 | governance/rules/rule_engine.py |
| 规则监控 | 执行统计 | governance/rules/rule_monitor.py |
| 规则生命周期 | 版本管理 | governance/rules/rule_lifecycle.py |

**规则特性**:
- 优先级调度: 按优先级执行规则
- 条件匹配: 支持复杂条件表达式
- 冷却时间: 防止规则重复触发
- 版本管理: 规则版本历史追踪

### 5. 自动化能力 (L6 Infrastructure)

| 模块 | 功能 | 文件 |
|------|------|------|
| 任务自动化器 | 队列管理 | infrastructure/automation/task_automator.py |
| 事件触发器 | 事件驱动 | infrastructure/automation/event_trigger.py |
| 智能调度器 | 周期调度 | infrastructure/automation/smart_scheduler.py |
| 流水线执行器 | 阶段管理 | infrastructure/automation/pipeline_executor.py |

**自动化特性**:
- 并发执行: 多线程任务处理
- 依赖管理: 任务依赖自动解析
- 资源管理: CPU/内存/IO资源控制
- 失败重试: 自动重试机制

## 门禁体系

### 检查项
1. **变更影响分析**: 分析代码变更影响范围
2. **层间依赖检查**: 验证六层架构依赖关系
3. **仓库完整性**: 检查文件结构和唯一真源
4. **JSON契约检查**: 验证JSON文件符合Schema
5. **技能安全检查**: 检查技能安全风险
6. **架构完整性**: 验证架构模块完整性

### 门禁模式
- **premerge**: 合并前检查
- **nightly**: 每日检查
- **release**: 发布检查
