# PROFILE_UPDATE_RULES.md - 画像更新规则

## 目的
定义哪些用户信息可进入画像，哪些不该进入，确保画像更新有治理。

## 适用范围
所有用户画像的更新操作。

## 信息分类

### 可进入画像
| 信息类型 | 说明 | 存储层级 |
|----------|------|----------|
| 长期稳定偏好 | 用户明确表达的长期偏好 | stable_preferences |
| 工作风格 | 用户的工作方式偏好 | working_style |
| 沟通风格 | 用户的沟通偏好 | communication_style |
| 长期目标 | 用户的长期目标 | long_term_goals |
| 重复任务 | 用户常做的任务 | recurring_tasks |
| 风险容忍度 | 用户的风险偏好 | risk_tolerance |
| 硬约束 | 用户明确约束 | saved_constraints |

### 仅留在会话层
| 信息类型 | 说明 | 原因 |
|----------|------|------|
| 临时偏好 | 当前会话有效 | 时效性短 |
| 一次性需求 | 仅本次需要 | 不具代表性 |
| 情绪状态 | 当前情绪 | 易变化 |
| 测试内容 | 测试目的 | 非真实偏好 |

### 禁止进入画像
| 信息类型 | 说明 | 原因 |
|----------|------|------|
| 敏感隐私 | 密码、密钥等 | 安全风险 |
| 他人信息 | 关于他人的信息 | 隐私保护 |
| 推断敏感 | 推断的敏感信息 | 准确性风险 |
| 临时状态 | 临时情绪/状态 | 不具代表性 |

## 偏好分类

### 长期稳定偏好
| 特征 | 说明 |
|------|------|
| 多次确认 | 用户多次表达 |
| 时间跨度 | 跨多个会话 |
| 明确表达 | 用户明确陈述 |
| 一致性 | 与其他偏好一致 |

### 阶段性偏好
| 特征 | 说明 |
|------|------|
| 项目相关 | 与特定项目相关 |
| 时间限定 | 有时间限制 |
| 可变化 | 可能随时间变化 |

### 一次性需求
| 特征 | 说明 |
|------|------|
| 单次有效 | 仅本次有效 |
| 特定场景 | 特定场景需求 |
| 不具代表性 | 不代表长期偏好 |

## 更新频率

### 立即更新
| 条件 | 说明 |
|------|------|
| 用户明确要求 | "记住我喜欢..." |
| 硬约束变更 | 用户约束变化 |
| 偏好冲突 | 新偏好与旧偏好冲突 |

### 批量更新
| 条件 | 说明 |
|------|------|
| 会话结束 | 会话结束时批量更新 |
| 定期同步 | 定期同步会话信息 |
| 用户确认 | 用户确认后更新 |

### 延迟更新
| 条件 | 说明 |
|------|------|
| 推断偏好 | 推断的偏好需确认 |
| 低置信度 | 置信度低的偏好 |
| 可能冲突 | 可能与现有偏好冲突 |

## 冲突处理

### 新旧偏好冲突
| 情况 | 处理方式 |
|------|----------|
| 新明确 > 旧推断 | 新偏好覆盖 |
| 新旧都明确 | 请求用户确认 |
| 新旧都推断 | 保留高置信度 |

### 偏好与约束冲突
| 情况 | 处理方式 |
|------|----------|
| 偏好与硬约束冲突 | 硬约束优先 |
| 偏好与软约束冲突 | 请求用户确认 |

## 撤销规则

### 可撤销
| 条件 | 说明 |
|------|------|
| 用户请求 | 用户明确要求撤销 |
| 错误记录 | 记录错误的信息 |
| 过期信息 | 已过期的信息 |

### 撤销流程
```
撤销请求
    ↓
┌─────────────────────────────────────┐
│ 1. 验证请求                          │
│    - 确认用户身份                    │
│    - 确认撤销内容                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行撤销                          │
│    - 标记为已撤销                    │
│    - 保留撤销记录                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 更新相关                          │
│    - 更新画像索引                    │
│    - 通知相关系统                    │
└─────────────────────────────────────┘
```

## 更新日志

### 日志格式
```json
{
  "updateId": "upd_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "type": "preference_update",
  "before": {
    "preference": "简洁回答",
    "confidence": 0.8
  },
  "after": {
    "preference": "详细回答",
    "confidence": 1.0,
    "source": "user_explicit"
  },
  "reason": "用户明确要求"
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 更新失败 | 保留旧值，记录错误 |
| 冲突无法解决 | 请求用户确认 |
| 撤销失败 | 记录错误，人工处理 |

## 维护方式
- 新增信息类型: 添加到信息分类表
- 调整更新频率: 更新更新频率表
- 新增冲突处理: 更新冲突处理表
