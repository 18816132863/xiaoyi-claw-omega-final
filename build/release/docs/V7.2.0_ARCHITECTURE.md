# V7.2.0 完整架构文档

## 一、六层架构总览

```
L1: Core              → 核心认知、身份、规则、标准
L2: Memory Context    → 记忆上下文、知识库、向量存储 (4096维)
L3: Orchestration     → 任务编排、工作流、恢复链
L4: Execution         → 能力执行、技能网关、生命周期
L5: Governance        → 稳定治理、安全审计、控制平面
L6: Infrastructure    → 基础设施、工具链、技能注册表
```

## 三、主链集成

### 1. skills 生命周期主链

```
InstallManager.install()
    ↓
SkillRegistry.register() → 写入 skill_registry.json
    ↓
EnableDisableManager.enable()/disable()
    ↓
DeprecationManager.deprecate()
    ↓
SkillRouter.select_skill()
    ├── registry.reload() 获取最新状态
    ├── 过滤 disabled/deprecated 技能
    ├── 接入 skill_budget_policy 预算检查
    └── 按 metrics 排序候选技能
```

### 2. workflow 恢复链主链

```
WorkflowEngine.run_workflow()
    ├── 注入 fallback_policy, rollback_manager, checkpoint_store
    │
    ├── _execute_dag()
    │   ├── step 前保存 checkpoint
    │   ├── _execute_step()
    │   │   ├── _execute_action() → 保留原始错误语义
    │   │   └── 失败时 FallbackPolicy.decide()
    │   │       ├── retry
    │   │       ├── fallback → fallback_used=True
    │   │       ├── skip
    │   │       └── abort
    │   └── step 后保存 checkpoint
    │
    └── WorkflowResult
        ├── failed_step_id
        ├── reason
        ├── retry_count
        ├── fallback_used
        ├── rollback_used
        └── checkpoint_id
```

### 3. metrics 反哺主链

```
generate_metrics.py
    ├── TaskSuccessBench → task_metrics.json
    ├── SkillLatencyBench → skill_metrics.json
    ├── MemoryRetrievalBench → memory_metrics.json
    └── EvaluationAggregator → aggregated_metrics.json

PolicyEngine.evaluate_policy()
    ├── _load_metrics() 加载 aggregated_metrics
    ├── _adjust_by_metrics() 基于指标调整决策
    │   ├── task_success_rate < 0.5 → degrade
    │   ├── skill_failure_rate > 0.3 → degrade
    │   └── overall_health == "poor" → degrade
    └── 返回 decision object

SkillRouter.__init__()
    └── _auto_load_metrics() 自动加载 metrics

SkillRouter.select_skill()
    ├── _check_metrics_reload() 检查 mtime 重新加载
    └── _sort_by_metrics() 按指标排序
        ├── failure_rate 高的排后
        └── avg_latency_ms 高的排后
```

## 四、核心接口

### PolicyEngine.evaluate_policy()

```python
def evaluate_policy(
    task_meta: Dict,
    profile: str,
    requested_capabilities: List[str] = None
) -> Dict:
    """
    统一策略评估入口
    
    接入组件：
    1. risk_classifier - 风险分类
    2. permission_engine - 权限检查
    3. token_budget_manager - Token 预算
    4. cost_budget_manager - 成本预算
    5. high_risk_guard - 高风险守卫
    6. metrics - 指标反哺
    
    Returns:
        {
            "decision": "allow" | "deny" | "degrade" | "review",
            "allowed": bool,
            "risk_level": str,
            "risk_score": float,
            "token_budget": int,
            "cost_budget": float,
            "allowed_capabilities": list,
            "blocked_capabilities": list,
            "requires_review": bool,
            "reason": str,
            "mitigations": list,
            "metrics": dict
        }
    """
```

### SkillRouter.select_skill()

```python
def select_skill(
    intent: str,
    profile: str,
    constraints: Dict = None,
    governance_decision: Dict = None
) -> Dict:
    """
    选择技能
    
    流程：
    1. _check_metrics_reload() - 检查并重新加载 metrics
    2. registry.reload() - 获取最新状态
    3. 按 intent 搜索候选
    4. 按 profile 过滤分类
    5. 按 status 过滤（排除 disabled/deprecated）
    6. 按 tags 过滤
    7. 按 governance_decision 过滤
    8. _sort_by_metrics() - 按 metrics 排序
    
    Returns:
        {
            "success": bool,
            "skill_id": str,
            "confidence": float,
            "reason": str,
            "profile": str,
            "category": str,
            "status": str,
            "tags": list
        }
    """
```

### WorkflowEngine.run_workflow()

```python
def run_workflow(
    workflow_spec: Dict,
    profile: str = "default",
    context_bundle: Dict = None,
    resume_from_checkpoint: str = None
) -> WorkflowResult:
    """
    执行工作流
    
    恢复链：
    - fallback_policy: retry/fallback/skip/abort
    - rollback_manager: 创建回滚点、执行回滚
    - checkpoint_store: 保存/加载检查点
    
    Returns:
        WorkflowResult:
            - workflow_id
            - status
            - step_results
            - failed_step_id
            - reason
            - retry_count
            - fallback_used
            - rollback_used
            - checkpoint_id
    """
```
