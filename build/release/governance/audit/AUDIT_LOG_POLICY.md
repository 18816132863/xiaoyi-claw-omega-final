# AUDIT_LOG_POLICY.md - 审计日志策略

## 目的
定义哪些行为必须写审计日志，确保关键动作有审计记录。

## 适用范围
所有需要审计的关键操作。

## 必须审计的行为

### 记忆写入
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 长期记忆写入 | 写入内容、来源、时间 | 1年 |
| 用户画像更新 | 更新内容、原因 | 1年 |
| 记忆删除 | 删除内容、原因 | 3年 |
| 记忆冲突处理 | 冲突内容、处理方式 | 1年 |

### 配置修改
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 系统配置修改 | 修改前后值、操作人 | 3年 |
| 规则文件修改 | 修改内容、原因 | 3年 |
| 边界配置修改 | 修改内容、原因 | 3年 |
| 技能配置修改 | 修改内容、原因 | 1年 |

### 规则变更
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 执行策略变更 | 变更内容、原因 | 3年 |
| 风险策略变更 | 变更内容、原因 | 3年 |
| 评测规则变更 | 变更内容、原因 | 1年 |
| 工作流变更 | 变更内容、原因 | 1年 |

### 高风险工具调用
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 删除操作 | 操作内容、确认记录 | 3年 |
| 批量操作 | 操作内容、范围 | 3年 |
| 系统命令执行 | 命令内容、结果 | 1年 |
| 外部API调用 | API内容、结果 | 1年 |

### 回滚操作
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 版本回滚 | 回滚原因、目标版本 | 3年 |
| 配置回滚 | 回滚原因、目标配置 | 3年 |
| 数据回滚 | 回滚原因、数据范围 | 3年 |

### 版本切换
| 行为 | 审计内容 | 保留期限 |
|------|----------|----------|
| 系统升级 | 升级版本、时间 | 3年 |
| 技能版本切换 | 切换内容、原因 | 1年 |
| 模型切换 | 切换内容、原因 | 1年 |

## 审计日志最小字段

### 必需字段
| 字段 | 类型 | 说明 |
|------|------|------|
| auditId | string | 审计ID |
| timestamp | datetime | 时间戳 |
| action | string | 操作类型 |
| actor | string | 操作者 |
| target | string | 操作对象 |
| result | string | 操作结果 |

### 可选字段
| 字段 | 类型 | 说明 |
|------|------|------|
| beforeValue | any | 操作前值 |
| afterValue | any | 操作后值 |
| reason | string | 操作原因 |
| sessionId | string | 会话ID |
| ipAddress | string | IP地址 |

## 审计日志格式

```json
{
  "auditId": "audit_20260406_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "action": "memory_write",
  "actor": {
    "type": "user",
    "id": "user_001"
  },
  "target": {
    "type": "long_term_memory",
    "id": "mem_001"
  },
  "details": {
    "beforeValue": null,
    "afterValue": {
      "key": "user_preference",
      "value": "简洁回答"
    },
    "reason": "用户明确要求"
  },
  "result": "success",
  "context": {
    "sessionId": "session_001",
    "channel": "xiaoyi-channel"
  }
}
```

## 审计日志保留策略

### 按行为类型
| 行为类型 | 保留期限 | 归档策略 |
|----------|----------|----------|
| 安全相关 | 3年 | 永久归档 |
| 配置变更 | 3年 | 压缩归档 |
| 记忆操作 | 1年 | 压缩归档 |
| 一般操作 | 90天 | 自动清理 |

### 按风险级别
| 风险级别 | 保留期限 | 说明 |
|----------|----------|------|
| CRITICAL | 永久 | 永久保留 |
| HIGH | 3年 | 长期保留 |
| MEDIUM | 1年 | 中期保留 |
| LOW | 90天 | 短期保留 |

## 审计日志查询

### 查询接口
| 参数 | 说明 |
|------|------|
| startTime | 开始时间 |
| endTime | 结束时间 |
| action | 操作类型 |
| actor | 操作者 |
| target | 操作对象 |
| result | 操作结果 |

### 查询示例
```json
{
  "query": {
    "startTime": "2026-04-01T00:00:00+08:00",
    "endTime": "2026-04-06T23:59:59+08:00",
    "action": "memory_write"
  },
  "pagination": {
    "offset": 0,
    "limit": 100
  }
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 日志写入失败 | 缓存后重试 |
| 存储空间不足 | 触发归档清理 |
| 查询超时 | 缩小查询范围 |

## 维护方式
- 新增审计行为: 添加到必须审计表
- 调整保留期限: 更新保留策略表
- 新增查询参数: 更新查询接口
