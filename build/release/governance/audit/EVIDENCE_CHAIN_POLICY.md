# EVIDENCE_CHAIN_POLICY.md - 证据链映射规则

**版本: V27.0**

## 目的
定义结论到证据链的映射规则，确保高价值输出可追证，实现"结论有据、证据可查、链路完整"。

## 适用范围
所有需要证据支撑的结论输出，包括：
- 分析报告
- 决策建议
- 事实陈述
- 推断结论
- 审计结论

---

## 一、证据链原则

| 原则 | 说明 |
|------|------|
| 结论有据 | 每个结论必须有证据支撑 |
| 证据可查 | 证据可追溯到原始来源 |
| 链路完整 | 从结论到源头链路完整 |
| 强度标注 | 证据强度必须标注 |
| 不确定性表达 | 不确定性必须明确表达 |

---

## 二、证据强度分级

### 2.1 强度等级

| 等级 | 名称 | 说明 | 可支撑程度 |
|------|------|------|------------|
| L1 | 确证 | 直接证据，无可辩驳 | 可独立支撑确定性结论 |
| L2 | 强证 | 多源一致，高度可信 | 可独立支撑高置信结论 |
| L3 | 中证 | 单源或多源部分一致 | 需配合其他证据 |
| L4 | 弱证 | 单一来源，可信度一般 | 仅作参考，需验证 |
| L5 | 推断 | 基于逻辑推断 | 必须标注为推断 |

### 2.2 证据类型与强度

| 证据类型 | 默认强度 | 说明 |
|----------|----------|------|
| 官方文档 | L1 | 政府、机构官方发布 |
| 系统日志 | L1 | 系统自动记录 |
| 审计记录 | L1 | 审计系统记录 |
| 专家证言 | L2 | 领域专家确认 |
| 多源一致 | L2 | 多个独立来源一致 |
| 单源数据 | L3 | 单一可靠来源 |
| 用户反馈 | L4 | 用户主观反馈 |
| 逻辑推断 | L5 | 基于已知推断未知 |

---

## 三、结论分类与证据要求

### 3.1 结论分类

| 分类 | 说明 | 最低证据要求 |
|------|------|--------------|
| 事实陈述 | 描述客观事实 | L3 单证或 L4 多证 |
| 分析结论 | 基于数据分析 | L2 强证 |
| 判断结论 | 基于专业判断 | L2 强证 + 专家确认 |
| 预测结论 | 预测未来趋势 | L3 + 明确不确定性 |
| 建议结论 | 提出行动建议 | L3 + 逻辑支撑 |
| 推断结论 | 基于推断 | L5 + 明确标注 |

### 3.2 证据数量要求

| 结论重要性 | 最低证据数量 | 推荐证据数量 |
|------------|--------------|--------------|
| 关键决策 | 3 条 L2+ | 5 条 L2+ |
| 重要结论 | 2 条 L3+ | 3 条 L3+ |
| 一般结论 | 1 条 L3+ | 2 条 L3+ |
| 参考信息 | 1 条 L4+ | 1 条 L4+ |

## 四、证据链结构

### 4.1 链路模型

```
结论 (Conclusion)
    │
    ├── 证据项 1 (Evidence Item)
    │       ├── 来源类型 (Source Type)
    │       ├── 来源引用 (Source Reference)
    │       ├── 来源强度 (Source Strength)
    │       ├── 提取内容 (Extracted Claims)
    │       └── 时效性 (Freshness)
    │
    ├── 证据项 2 (Evidence Item)
    │       └── ...
    │
    └── 证据项 N (Evidence Item)
            └── ...
```

### 4.2 证据链数据结构

```json
{
  "conclusion_id": "conc_001",
  "conclusion_text": "系统性能提升 50%",
  "conclusion_type": "fact",

    {
      "evidence_id": "ev_001",
      "source_type": "system_log",
      "source_ref": "logs/perf_20260408.log",
      "source_strength": "L1",
      "extracted_claims": [
        "响应时间从 100ms 降至 50ms",
        "吞吐量从 1000 QPS 提升至 2000 QPS"

}
```

## 五、事实与推断区分

### 5.1 区分规则

| 类型 | 特征 | 标注要求 |
|------|------|----------|
| 事实 | 可验证、有直接证据 | 标注证据来源 |
| 推断 | 基于逻辑、间接得出 | 标注推断依据 + 不确定性 |
| 假设 | 假设条件 | 标注为假设 + 条件说明 |
| 观点 | 主观看法 | 标注为观点 + 持有者 |

### 5.2 推断标注规范

```
推断结论格式：
[推断] {结论内容}
依据：{推断依据}
不确定性：{高/中/低}
验证建议：{如何验证}
```

### 5.3 示例

```
✅ 正确：
[事实] 系统响应时间从 100ms 降至 50ms
证据：系统日志 logs/perf_20260408.log L1

✅ 正确：
[推断] 用户满意度可能提升
依据：响应时间降低通常提升满意度
不确定性：中
验证建议：进行用户满意度调查

❌ 错误：
用户满意度提升（无证据、无标注）
```

## 六、低强度证据使用规则

### 6.1 使用限制

| 证据强度 | 可独立使用 | 需配合使用 | 禁止使用场景 |
|----------|------------|------------|--------------|
| L1 确证 | ✅ 是 | - | - |
| L2 强证 | ✅ 是 | - | - |
| L3 中证 | ⚠️ 一般结论 | 关键/重要结论 | - |
| L4 弱证 | ❌ 否 | 需 L3+ 配合 | 关键决策 |
| L5 推断 | ❌ 否 | 需明确标注 | 事实陈述 |

### 6.2 低强度证据支撑程度

| 证据强度 | 可支撑结论类型 | 置信度上限 |
|----------|----------------|------------|
| L4 单证 | 参考信息 | 60% |
| L4 多证 | 一般结论 | 75% |
| L5 推断 | 推断结论 | 50% |

### 6.3 低强度证据使用声明

```
使用低强度证据时必须声明：
"本结论基于 [L4/L5] 证据，置信度 [XX%]，建议 [验证方式]"
```

## 七、不确定性表达要求

### 7.1 不确定性等级

| 等级 | 置信度范围 | 表达方式 |
|------|------------|----------|
| 确定 | > 95% | "确定"、"确认" |
| 高置信 | 80-95% | "很可能"、"高度可信" |
| 中置信 | 60-80% | "可能"、"倾向于" |
| 低置信 | 40-60% | "或许"、"有一定可能" |
| 不确定 | < 40% | "不确定"、"需验证" |

### 7.2 不确定性标注

| 场景 | 标注要求 |
|------|----------|
| 事实陈述 | 置信度 > 90% 或标注不确定性 |
| 分析结论 | 必须标注置信度 |
| 预测结论 | 必须标注置信区间 |
| 推断结论 | 必须标注推断依据和不确定性 |

---

## 八、证据链验证

### 8.1 验证项

| 验证项 | 说明 | 通过标准 |
|--------|------|----------|
| 完整性 | 证据链是否完整 | 所有结论有证据 |
| 一致性 | 证据是否一致 | 无矛盾证据 |
| 时效性 | 证据是否有效 | 无过期证据 |
| 强度匹配 | 强度是否匹配结论 | 满足最低要求 |
| 来源可信 | 来源是否可信 | 来源可靠 |

### 8.2 验证流程

```
证据链生成
    ↓
┌─────────────────────────────────────┐
│ 1. 完整性检查                        │
│    - 每个结论是否有证据              │
│    - 证据引用是否有效                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 一致性检查                        │
│    - 证据之间是否矛盾                │
│    - 矛盾如何处理                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 强度匹配检查                      │
│    - 证据强度是否满足结论要求        │
│    - 不满足时如何处理                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 时效性检查                        │
│    - 证据是否过期                    │
│    - 过期证据如何处理                │
└─────────────────────────────────────┘
    ↓
验证结果输出
```

## 九、证据链存储与查询

### 9.1 存储要求

| 存储项 | 说明 | 保留期限 |
|--------|------|----------|
| 结论 | 结论内容 | 与输出同生命周期 |
| 证据链 | 证据链结构 | 与输出同生命周期 |
| 原始证据 | 原始证据数据 | 按证据类型策略 |
| 验证记录 | 验证过程记录 | 90 天 |

### 9.2 查询接口

| 接口 | 说明 |
|------|------|
| 按结论查证据 | 查询支撑某结论的证据链 |
| 按证据查结论 | 查询某证据支撑的所有结论 |
| 验证状态查询 | 查询证据链验证状态 |
| 完整性报告 | 生成证据链完整性报告 |

---

## 十、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 证据缺失 | 标注"证据缺失" + 降级处理 |
| 证据矛盾 | 标注矛盾 + 选择更强证据 |
| 证据过期 | 标注"证据过期" + 建议更新 |
| 强度不足 | 标注"证据强度不足" + 降置信度 |
| 来源失效 | 标注"来源失效" + 验证失败 |

---

## 十一、完成标准

| 标准 | 验证方式 |
|------|----------|
| 结论有据 | 所有结论有证据支撑 |
| 证据可查 | 证据可追溯到源头 |
| 链路完整 | 验证通过率 > 95% |
| 强度匹配 | 强度匹配率 > 90% |
| 不确定性表达 | 不确定性标注率 100% |
