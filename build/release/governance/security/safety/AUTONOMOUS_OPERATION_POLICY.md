# AUTONOMOUS_OPERATION_POLICY.md - 自主操作策略

## 目的
定义自主操作的执行策略，确保用户只需下达一次命令，系统自动完成所有操作。

## 适用范围
所有手机操作、软件互联、跨平台对接场景。

---

## 一、自主操作原则

### 1.1 核心原则
| 原则 | 说明 |
|------|------|
| 一次指令 | 用户只需下达一次命令 |
| 零确认 | 常规操作无需确认 |
| 智能决策 | 系统自主判断最优路径 |
| 异常自愈 | 自动尝试替代方案 |
| 结果反馈 | 完成后主动汇报 |

### 1.2 信任分级
| 级别 | 操作类型 | 执行策略 |
|------|----------|----------|
| **完全信任** | 读取、搜索、查看 | 自动执行，无需汇报 |
| **高信任** | 创建、发送、设置 | 自动执行，事后汇报 |
| **中信任** | 修改、更新 | 首次确认后自动 |
| **低信任** | 删除、拨打 | 学习信任度后自动 |

## 三、自主操作流程

### 3.1 标准流程
```
用户指令
    ↓
┌─────────────────────────────────────┐
│ 1. 意图理解                          │
│    - 自动分析用户意图                │
│    - 提取操作参数                    │
│    - 分解操作步骤                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 信任评估                          │
│    - 判断操作信任级别                │
│    - 确定执行策略                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 自主执行                          │
│    - 按步骤执行操作                  │
│    - 异常自动恢复                    │
│    - 状态实时追踪                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 结果汇报                          │
│    - 执行结果摘要                    │
│    - 异常情况说明                    │
│    - 后续建议（如有）                │
└─────────────────────────────────────┘
```

### 3.2 异常处理流程
```
操作异常
    ↓
┌─────────────────────────────────────┐
│ 1. 异常识别                          │
│    - 识别异常类型                    │
│    - 评估恢复可能性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 自动恢复                          │
│    - 尝试替代方案                    │
│    - 重试操作                        │
│    - 降级处理                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 结果处理                          │
│    - 恢复成功 → 继续执行             │
│    - 恢复失败 → 汇报并建议           │
└─────────────────────────────────────┘
```

## 五、安全边界

### 5.1 仍需确认的操作
| 操作 | 原因 |
|------|------|
| 财务操作 | 涉及资金安全 |
| 删除大量数据 | 不可逆操作 |
| 系统设置修改 | 影响系统稳定性 |
| 敏感信息发送 | 隐私保护 |

### 5.2 禁止自动执行
| 操作 | 原因 |
|------|------|
| 支付操作 | 资金安全 |
| 密码修改 | 账户安全 |
| 权限授予 | 安全风险 |
| 紧急呼叫 | 误操作风险 |

## 六、汇报模板

### 6.1 成功汇报
```
✅ 已完成：{操作名称}
   详情：{操作详情}
   耗时：{执行时间}秒
```

### 6.2 部分成功汇报
```
⚠️ 部分完成：{操作名称}
   成功：{成功数量}
   失败：{失败数量}
   原因：{失败原因}
```

### 6.3 失败汇报
```
❌ 执行失败：{操作名称}
   原因：{失败原因}
   已尝试：{恢复尝试}
   建议：{解决建议}
```
