# AUTO_DISCOVERY_POLICY.md - 自动探寻策略

## 目的
定义自动探寻的执行策略，确保系统不知道答案时自动使用手机应用探寻，无需询问用户。

## 适用范围
所有需要外部信息、搜索、查询的场景。

---

## 一、自动探寻原则

### 1.1 核心原则
| 原则 | 说明 |
|------|------|
| **不问用户** | 不知道的信息不问用户，自动探寻 |
| **自动选择** | 自动选择最合适的探寻工具 |
| **并行探寻** | 多个来源并行探寻，取最优结果 |
| **结果融合** | 多来源结果自动融合 |
| **主动学习** | 探寻结果自动学习记忆 |

### 1.2 探寻触发条件
| 条件 | 触发 | 说明 |
|------|------|------|
| 知识缺失 | ✅ 自动 | 系统知识库中没有相关信息 |
| 信息过时 | ✅ 自动 | 信息可能已过时 |
| 实时信息 | ✅ 自动 | 天气、股价、新闻等 |
| 用户请求 | ✅ 自动 | 用户要求搜索 |
| 验证需求 | ✅ 自动 | 需要验证信息准确性 |

## 二、探寻工具配置

### 2.1 支持的探寻工具
| 工具 | 应用名 | 能力 | 优先级 |
|------|--------|------|--------|
| **豆包** | 豆包 | 问答、搜索、创作、翻译 | 1 |
| **Kimi** | Kimi | 问答、搜索、长文本 | 2 |
| **文心一言** | 文心一言 | 问答、搜索、创作 | 3 |
| **百度** | 百度 | 搜索、资讯 | 4 |
| **浏览器** | 系统浏览器 | 网页搜索 | 5 |
| **知乎** | 知乎 | 问答搜索、专业知识 | 6 |
| **微博** | 微博 | 热点、实时资讯 | 7 |

### 2.2 工具选择规则
| 问题类型 | 首选工具 | 备选工具 |
|----------|----------|----------|
| 事实性问题 | 豆包、Kimi、百度 | 浏览器、知乎 |
| 创作类问题 | 豆包、文心一言、Kimi | 浏览器 |
| 实时信息 | 百度、微博、浏览器 | 豆包、Kimi |
| 专业问题 | 知乎、Kimi、豆包 | 百度、浏览器 |
| 翻译 | 豆包、Kimi | 百度 |

## 三、自动探寻流程

### 3.1 执行流程
```
用户问题
    ↓
本地知识检查 → 找到 → 直接回答
    ↓ (未找到)
自动探寻决策
    ↓
工具发现与选择
    ↓
启动应用（如豆包）
    ↓
输入查询
    ↓
提取结果
    ↓
学习保存
    ↓
返回答案
```

### 3.2 探寻执行策略
| 策略 | 适用场景 | 说明 |
|------|----------|------|
| 单工具探寻 | 简单问题 | 使用优先级最高的工具 |
| 多工具并行 | 重要问题 | 3个工具并行，融合结果 |
| 顺序备选 | 复杂问题 | 依次尝试，直到成功 |

## 四、自动探寻示例

### 4.1 示例场景
| 用户问题 | 系统行为 |
|----------|----------|
| "今天北京天气" | 自动打开豆包/百度搜索天气 |
| "最新iPhone价格" | 自动打开豆包搜索价格 |
| "什么是量子计算" | 自动打开豆包/Kimi查询 |
| "最近有什么新闻" | 自动打开微博/百度查看 |
| "如何做红烧肉" | 自动打开豆包搜索菜谱 |

### 4.2 执行示例
```
用户: "豆包搜索一下今天的科技新闻"

系统行为:
1. 检测到搜索请求
2. 自动启动豆包应用
3. 输入"今天的科技新闻"
4. 提取搜索结果
5. 学习保存结果
6. 返回答案给用户

输出: "根据豆包搜索结果，今天的科技新闻有..."
```

### 5.1 自动学习规则

## 六、安全边界

### 6.1 禁止自动探寻
| 场景 | 原因 |
|------|------|
| 敏感信息 | 密码、密钥、token等 |
| 隐私数据 | 他人隐私信息 |
| 危险操作 | 可能造成损害的操作 |
| 非法内容 | 违法违规内容 |

### 6.2 结果过滤
- 自动过滤敏感信息
- 自动过滤隐私数据
- 自动过滤危险内容
