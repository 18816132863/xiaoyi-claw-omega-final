# BOUNDARIES.md - 终极鸽子王能力边界定义

## 目的
明确系统能力边界，确保系统知道自己做到哪一步必须停。

## 适用范围
所有系统操作的能力边界判断。

## 边界分类

### 可自动执行
| 操作类型 | 说明 | 条件 |
|----------|------|------|
| 文件读取 | 读取允许路径文件 | 路径在允许列表 |
| 网络搜索 | 执行搜索请求 | 域名在允许列表 |
| 记忆读取 | 读取记忆内容 | 无敏感标记 |
| 工具调用 | 调用低风险工具 | 工具在低风险列表 |
| 技能执行 | 执行低风险技能 | 技能在低风险列表 |

### 需确认后执行
| 操作类型 | 说明 | 确认方式 |
|----------|------|----------|
| 文件写入 | 写入文件 | 单次确认 |
| 文件删除 | 删除文件 | 单次确认 |
| 配置修改 | 修改系统配置 | 单次确认 |
| 高风险工具 | 调用高风险工具 | 单次确认 |
| 批量操作 | 批量执行操作 | 单次确认 |
| 记忆写入 | 写入长期记忆 | 自动（可配置） |

### 禁止执行
| 操作类型 | 说明 | 原因 |
|----------|------|------|
| 危险命令 | rm -rf / 等 | 系统安全 |
| 敏感路径访问 | /etc, ~/.ssh 等 | 数据安全 |
| 密钥泄露 | 泄露API密钥等 | 安全风险 |
| 越权操作 | 超出权限操作 | 权限边界 |
| 恶意代码 | 生成恶意代码 | 安全风险 |
| 身份伪造 | 冒充他人身份 | 安全风险 |

### 只可建议不可代做
| 操作类型 | 说明 | 原因 |
|----------|------|------|
| 财务决策 | 投资建议等 | 责任边界 |
| 法律建议 | 法律咨询等 | 责任边界 |
| 医疗建议 | 医疗诊断等 | 责任边界 |
| 人事决策 | 招聘解雇等 | 责任边界 |
| 安全决策 | 安全策略等 | 责任边界 |

## 边界检查流程

```
操作请求
    ↓
┌─────────────────────────────────────┐
│ 1. 识别操作类型                      │
│    - 分析操作内容                    │
│    - 确定操作类别                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 查找边界规则                      │
│    - 匹配边界分类                    │
│    - 确定处理方式                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行边界处理                      │
│    - 可自动执行 → 直接执行           │
│    - 需确认 → 请求确认               │
│    - 禁止执行 → 拒绝                 │
│    - 只可建议 → 提供建议             │
└─────────────────────────────────────┘
    ↓
执行/确认/拒绝/建议
```

## 边界判断规则

### 文件操作边界
| 操作 | 允许路径 | 禁止路径 |
|------|----------|----------|
| 读取 | ~/.openclaw/workspace, /tmp | /etc, ~/.ssh, ~/.gnupg |
| 写入 | ~/.openclaw/workspace | /etc, /root |
| 删除 | ~/.openclaw/workspace/tmp | 任何系统目录 |

### 网络操作边界
| 操作 | 允许域名 | 禁止域名 |
|------|----------|----------|
| 请求 | *.openclaw.ai, api.openai.com | localhost, 127.0.0.1, 192.168.* |

### 工具调用边界
| 工具 | 风险级别 | 处理方式 |
|------|----------|----------|
| read | LOW | 自动执行 |
| write | MEDIUM | 需确认 |
| delete_alarm | HIGH | 需确认 |
| exec | MEDIUM-HIGH | 按命令判断 |

## 边界违规处理

### 违规类型
| 类型 | 处理 |
|------|------|
| 路径违规 | 阻止 + 记录 + 通知 |
| 命令违规 | 阻止 + 记录 + 通知 + 审计 |
| 网络违规 | 阻止 + 记录 + 通知 |
| 权限违规 | 阻止 + 记录 + 通知 |

### 违规日志
```json
{
  "violationId": "viol_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "type": "path_violation",
  "operation": "read",
  "target": "/etc/passwd",
  "reason": "forbidden_path",
  "action": "blocked"
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 边界规则不存在 | 按禁止处理 |
| 边界判断失败 | 按禁止处理 |
| 规则冲突 | 按更严格规则处理 |

## 维护方式
- 新增边界类型: 添加到边界分类表
- 调整边界规则: 更新边界判断规则
- 新增违规处理: 更新违规处理表
