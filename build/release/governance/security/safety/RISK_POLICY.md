# RISK_POLICY.md - 终极鸽子王风险策略

## 目的
定义操作风险分级和对应处理策略，确保高风险操作得到适当控制。

## 适用范围
所有工具调用、技能执行、系统操作。

## 风险分级

### 风险等级定义
| 等级 | 名称 | 风险值 | 说明 |
|------|------|--------|------|
| LOW | 低风险 | 1 | 可自动执行，无副作用 |
| MEDIUM | 中风险 | 2 | 需记录，可恢复副作用 |
| HIGH | 高风险 | 3 | 需确认，不可逆操作 |
| CRITICAL | 极高风险 | 4 | 需双重确认，系统级操作 |

### 操作风险映射
| 操作类型 | 风险等级 | 原因 |
|----------|----------|------|
| 文件读取 | LOW | 无副作用 |
| 网络搜索 | LOW | 无副作用 |
| 文件写入 | MEDIUM | 可覆盖 |
| 文件删除 | HIGH | 不可逆 |
| 配置修改 | HIGH | 影响系统 |
| 技能安装 | MEDIUM | 可卸载 |
| 技能卸载 | HIGH | 数据丢失风险 |
| 系统重启 | CRITICAL | 服务中断 |
| 数据清空 | CRITICAL | 不可恢复 |

## 风险检查流程

```
操作请求
    ↓
┌─────────────────────────────────────┐
│ 1. 识别操作类型                      │
│ 2. 查询风险等级                      │
│ 3. 计算累积风险                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ LOW: 直接执行                        │
│ MEDIUM: 记录日志后执行               │
│ HIGH: 请求用户确认                   │
│ CRITICAL: 双重确认 + 审计            │
└─────────────────────────────────────┘
    ↓
执行/拒绝
```

## 累积风险计算

### 计算公式
```
cumulativeRisk = Σ(operationRisk × operationCount)
```

### 累积风险阈值
| 阈值 | 动作 |
|------|------|
| < 5 | 正常执行 |
| 5-10 | 需要确认 |
| > 10 | 需要审批 |

## 确认策略

### 单次确认
- 风险等级 HIGH
- 累积风险 5-10
- 首次执行高风险操作

### 双重确认
- 风险等级 CRITICAL
- 累积风险 > 10
- 批量删除操作
- 系统配置修改

### 确认超时
- 确认等待时间: 60秒
- 超时动作: 取消操作
- 超时记录: 记入审计日志

## 审计要求

### 审计触发条件
- 风险等级 ≥ HIGH
- 累积风险 > 10
- 用户拒绝确认
- 操作失败

### 审计记录内容
```json
{
  "auditId": "audit_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "operation": "file_delete",
  "riskLevel": "HIGH",
  "userId": "user_001",
  "confirmed": true,
  "result": "success"
}
```

## 风险降级

### 自动降级
| 条件 | 降级方案 |
|------|----------|
| 用户拒绝确认 | 跳过操作 |
| 确认超时 | 取消操作 |
| 风险不可接受 | 拒绝执行 |

### 替代方案
| 高风险操作 | 低风险替代 |
|------------|------------|
| 文件删除 | 移动到回收站 |
| 配置覆盖 | 创建备份后修改 |
| 批量操作 | 单个操作 + 确认 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 风险等级未知 | 按 HIGH 处理 |
| 累积风险超限 | 强制确认 |
| 审计失败 | 阻止操作 |

## 维护方式
- 新增操作类型: 更新风险映射表
- 调整风险等级: 修改等级定义
- 调整阈值: 修改累积风险阈值
