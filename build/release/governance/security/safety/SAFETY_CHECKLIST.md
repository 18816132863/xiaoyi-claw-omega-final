# SAFETY_CHECKLIST.md - 安全检查清单

## 目的
定义操作前的安全检查项，确保操作安全。

## 适用范围
所有高风险操作前的安全验证。

## 检查清单

### 文件操作
| 检查项 | 通过条件 | 风险级别 |
|--------|----------|----------|
| 路径允许 | 路径在允许列表 | HIGH |
| 非关键文件 | 非系统关键文件 | MEDIUM |
| 大小合理 | 文件大小 < 限制 | LOW |
| 有备份 | 已创建备份 | MEDIUM |

### 网络操作
| 检查项 | 通过条件 | 风险级别 |
|--------|----------|----------|
| 域名允许 | 域名在允许列表 | HIGH |
| 非内网 | 非内网地址 | HIGH |
| HTTPS | 使用HTTPS | MEDIUM |
| 证书有效 | SSL证书有效 | MEDIUM |

### 命令执行
| 检查项 | 通过条件 | 风险级别 |
|--------|----------|----------|
| 命令允许 | 命令在允许列表 | CRITICAL |
| 非危险命令 | 非危险命令 | CRITICAL |
| 权限足够 | 有执行权限 | HIGH |

### 记忆操作
| 检查项 | 通过条件 | 风险级别 |
|--------|----------|----------|
| 无敏感数据 | 不含敏感信息 | HIGH |
| 容量足够 | 存储空间足够 | LOW |
| 格式正确 | 格式符合规范 | MEDIUM |

## 检查流程

```
操作请求
    ↓
┌─────────────────────────────────────┐
│ 1. 识别操作类型                      │
│    - 文件/网络/命令/记忆              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 加载对应检查清单                  │
│    - 获取检查项                      │
│    - 确定风险级别                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行检查                          │
│    - 逐项验证                        │
│    - 记录结果                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 输出检查结果                      │
│    - 通过/不通过                    │
│    - 风险评估                        │
└─────────────────────────────────────┘
```

## 检查报告

### 报告格式
```json
{
  "checkId": "check_20260406_001",
  "timestamp": "2026-04-06T12:13:00+08:00",
  "operation": "file_write",
  "target": "/path/to/file",
  "checks": [
    {
      "item": "path_allowed",
      "result": "pass",
      "riskLevel": "HIGH"
    },
    {
      "item": "not_critical",
      "result": "pass",
      "riskLevel": "MEDIUM"
    }
  ],
  "overallResult": "pass",
  "riskLevel": "MEDIUM"
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 检查失败 | 阻止操作 |
| 风险过高 | 需确认 |
| 检查超时 | 使用默认 |
