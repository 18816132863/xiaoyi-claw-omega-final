# 沙箱隔离策略

**版本**: 1.0.0  
**创建时间**: 2026-04-07 00:05 UTC

---

## 🎯 目的

防止测试代码、新插件、实验性功能污染核心架构。

---

## 🏗️ 隔离架构

```
┌─────────────────────────────────────────────────────────┐
│                    核心架构 (CORE)                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │  runtime/  safety/  governance/  memory/        │    │
│  │  只读保护，任何修改需审批                          │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
                          ↑
                          │ 只读访问
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    沙箱层 (SANDBOX)                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │  测试沙箱     │  │  插件沙箱     │  │  实验沙箱     │  │
│  │  sandbox/    │  │  plugins/    │  │  experiments/ │  │
│  │  test/       │  │  external/   │  │  lab/        │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│                                                         │
│  - 完全隔离，无法修改核心文件                              │
│  - 失败不影响核心                                        │
│  - 可随时删除/重置                                       │
└─────────────────────────────────────────────────────────┘
```

## 🛡️ 保护规则

### 1. 核心保护
```
规则: 核心文件只读
实施:
  - 核心目录标记为 PROTECTED
  - 任何修改需用户明确审批
  - 自动备份修改前的文件
```

### 2. 沙箱隔离
```
规则: 沙箱完全隔离
实施:
  - 沙箱无法直接修改核心
  - 沙箱通过 API 访问核心
  - 沙箱失败不影响核心
```

### 3. 插件审核
```
规则: 插件需审核才能安装
实施:
  - 新插件进入 pending/
  - 安全扫描 + 功能测试
  - 审批后进入 installed/
  - 有问题移入 quarantine/
```

---

## 📋 操作流程

### 测试新功能
```
1. 在 sandbox/test/ 创建测试环境
2. 运行测试，验证功能
3. 测试通过 → 申请合并到核心
4. 用户审批 → 合并
5. 测试失败 → 删除沙箱，核心不受影响
```

### 安装插件
```
1. 插件放入 plugins/pending/
2. 自动安全扫描
3. 在 sandbox/ 中测试
4. 测试通过 → 用户审批
5. 审批通过 → 移入 plugins/installed/
6. 发现问题 → 移入 plugins/quarantine/
```

### 实验新架构
```
1. 在 experiments/lab/ 创建实验环境
2. 自由实验，不影响核心
3. 实验成功 → 申请合并
4. 实验失败 → 删除或归档
```

## ⚠️ 污染检测

| 检测项 | 方法 | 频率 |
|--------|------|------|
| 核心文件完整性 | Hash校验 | 每次操作 |
| 未授权修改 | 文件监控 | 实时 |
| 恶意代码 | 静态分析 | 安装时 |
| 资源泄露 | 监控 | 实时 |

---

## 🔄 回滚机制

```
污染发生 → 立即隔离 → 备份当前 → 恢复备份 → 验证完整性 → 记录审计
```
