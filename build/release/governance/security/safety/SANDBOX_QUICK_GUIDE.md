# 沙箱使用指南

**版本**: 1.0.0  
**创建时间**: 2026-04-07 00:05 UTC

---

## 🚀 快速开始

### 测试新功能
```
1. 告诉我你想测试什么
2. 我会在 sandbox/ 创建隔离环境
3. 在沙箱中运行测试
4. 测试完成，给你报告
5. 你决定是否合并到核心
```

### 安装新插件
```
1. 告诉我插件信息
2. 我会放入 plugins/pending/
3. 自动安全扫描
4. 在沙箱中测试
5. 给你安全报告
6. 你决定是否安装
```

### 实验新架构
```
1. 告诉我你的想法
2. 我会在 experiments/lab/ 创建环境
3. 自由实验，不影响核心
4. 实验完成，给你报告
5. 你决定下一步
```

---

## 📁 沙箱目录

| 目录 | 用途 | 风险 |
|------|------|------|
| `sandbox/test/` | 测试代码 | 低，完全隔离 |
| `sandbox/staging/` | 预发布验证 | 低，模拟数据 |
| `plugins/pending/` | 待审批插件 | 中，需扫描 |
| `plugins/installed/` | 已安装插件 | 低，已审核 |
| `experiments/lab/` | 实验环境 | 低，完全隔离 |

---

## 🛡️ 安全保证

| 保证 | 说明 |
|------|------|
| ✅ 核心不污染 | 沙箱无法修改核心文件 |
| ✅ 失败可恢复 | 沙箱失败可随时删除 |
| ✅ 数据不泄露 | 沙箱使用模拟数据 |
| ✅ 操作可审计 | 所有操作都有记录 |

## 📋 操作示例

### 示例1: 测试新技能
```
用户: 我想测试一个新技能
我: 
  1. 创建 sandbox/test/new-skill/
  2. 在沙箱中运行测试
  3. 报告测试结果
  4. 等待你的决定
```

### 示例2: 安装外部插件
```
用户: 我想安装 xxx 插件
我:
  1. 下载到 plugins/pending/
  2. 运行安全扫描
  3. 在沙箱中测试功能
  4. 生成安全报告
  5. 等待你的审批
```

### 示例3: 实验新架构
```
用户: 我想尝试新的记忆架构
我:
  1. 创建 experiments/lab/memory-v2/
  2. 实现新架构
  3. 运行对比测试
  4. 报告优缺点
  5. 等待你的决定
```

## ⚠️ 注意事项

1. **沙箱是临时的** - 不想保留的内容放沙箱，随时可删
2. **核心是永久的** - 核心文件修改需你明确审批
3. **插件需审核** - 外部插件必须经过安全扫描
4. **实验有风险** - 实验性功能可能不稳定

---

## 🔄 清理沙箱

```
命令: "清理沙箱"
效果: 删除所有沙箱内容，核心不受影响
```
