# AUTO_OPS_GUARDRAILS.md - 自动运维护栏

**版本: V27.0**

## 目的
定义自动运维护栏，确保自动运维能力有边界，不会越修越乱，实现"边界明确、权限受控、可熔断、可审计"。

## 适用范围
所有自动运维场景，包括：
- 权限边界
- 区域边界
- 租户边界
- 服务边界
- 审批边界
- 熔断与 kill switch

---

## 一、护栏原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| 默认禁止 | 无明确授权则禁止自动运维 | 白名单机制 |
| 边界明确 | 所有边界必须明确定义 | 边界配置 |
| 最小权限 | 只授予必要的最小权限 | RBAC |
| 可熔断 | 随时可熔断停止 | 熔断机制 |
| 可审计 | 所有操作可审计 | 审计日志 |
| 可回滚 | 操作可回滚 | 回滚机制 |
| 可预测 | 操作结果可预测 | 预检查 |
| 可追溯 | 操作链路可追溯 | 链路追踪 |

## 二、权限边界

### 2.1 权限分级

| 级别 | 代码 | 说明 | 允许操作 | 风险等级 |
|------|------|------|----------|----------|
| L0-只读 | READ_ONLY | 只读权限 | 查询、监控 | 无风险 |
| L1-诊断 | DIAGNOSE | 诊断权限 | 查询、日志、分析 | 低风险 |
| L2-轻量修复 | LIGHT_FIX | 轻量修复权限 | 重启服务、清理缓存 | 低风险 |
| L3-中度修复 | MEDIUM_FIX | 中度修复权限 | 配置修改、资源调整 | 中风险 |
| L4-重度修复 | HEAVY_FIX | 重度修复权限 | 数据修改、架构变更 | 高风险 |
| L5-管理员 | ADMIN | 管理员权限 | 所有操作 | 最高风险 |

| 查看日志 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 查看配置 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 执行诊断 | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 分析性能 | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 重启服务 | ❌ | ❌ | ✅ | ✅ | ✅ | ✅ |
| 清理缓存 | ❌ | ❌ | ✅ | ✅ | ✅ | ✅ |
| 重置连接 | ❌ | ❌ | ✅ | ✅ | ✅ | ✅ |
| 修改配置 | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ |
| 调整资源 | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ |
| 扩缩容 | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ |
| 修改数据 | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |

| 跨租户操作 | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |

### 2.3 权限授予

| 授予方式 | 说明 | 审批要求 | 有效期 |
|----------|------|----------|--------|
| 角色继承 | 从角色继承权限 | 无 | 永久 |
| 显式授予 | 显式授予权限 | 服务Owner审批 | 永久 |
| 临时授权 | 临时授予权限 | 服务Owner审批 | 24小时 |
| 紧急授权 | 紧急情况授权 | 事后补审批 | 4小时 |
| 自动授权 | 自动修复场景授权 | 预审批 | 单次 |

## 三、区域边界

### 3.1 区域隔离

| 边界规则 | 说明 | 检查方式 |
|----------|------|----------|
| 单区域操作 | 自动运维仅限单区域 | 区域标签检查 |
| 跨区禁止 | 禁止跨区域自动运维 | 跨区检测 |
| 区域白名单 | 只有白名单区域允许自动运维 | 白名单检查 |
| 区域状态 | 检查区域当前状态 | 状态检查 |

### 3.2 区域配置

| 区域 | 自动运维 | 允许操作 | 审批要求 | 说明 |
|------|----------|----------|----------|------|
| 生产-核心 | 禁止 | NONE | 禁止 | 核心生产区域 |
| 生产-标准 | 受限 | LIGHT_FIX | 服务Owner | 标准生产区域 |
| 生产-边缘 | 允许 | MEDIUM_FIX | 无 | 边缘生产区域 |
| 预发 | 允许 | MEDIUM_FIX | 无 | 预发环境 |
| 测试 | 允许 | HEAVY_FIX | 无 | 测试环境 |
| 开发 | 允许 | HEAVY_FIX | 无 | 开发环境 |
| DR站点 | 受限 | LIGHT_FIX | 技术负责人 | 容灾站点 |

### 3.3 跨区规则

| 规则 | 说明 | 审批要求 |
|------|------|----------|
| 禁止自动跨区 | 禁止自动执行跨区操作 | 禁止 |
| 审批后跨区 | 审批后可执行跨区操作 | 技术总监 |
| 影响评估 | 跨区前必须评估影响 | 必须 |
| 通知相关方 | 跨区前通知相关区域 | 必须 |

## 七、熔断机制

### 7.1 熔断触发条件

| 条件 | 说明 | 阈值 | 熔断级别 |
|------|------|------|----------|
| 失败率高 | 失败率过高 | > 30% | 服务级 |
| 影响扩大 | 导致新故障 | 检测到新故障 | 全局 |
| 连续失败 | 连续失败次数过多 | > 3 次 | 操作级 |
| 人工请求 | 人工请求停止 | 人工触发 | 全局 |
| 系统异常 | 系统检测异常 | 异常检测 | 全局 |
| 外部事件 | 外部重大事件 | 事件触发 | 全局 |
| 边界违规 | 边界检查失败 | 检测到违规 | 操作级 |

### 7.2 熔断级别

| 级别 | 代码 | 说明 | 影响范围 | 恢复审批 |
|------|------|------|----------|----------|
| L1-操作级 | OPERATION | 停止特定操作 | 单个操作 | 服务Owner |
| L2-服务级 | SERVICE | 停止服务相关操作 | 单个服务 | 技术负责人 |
| L3-区域级 | REGION | 停止区域相关操作 | 单个区域 | 技术总监 |
| L4-全局 | GLOBAL | 停止所有自动运维 | 全平台 | CTO |

### 7.3 熔断动作

```
熔断触发
    ↓
┌─────────────────────────────────────┐
│ 1. 停止操作                          │
│    - 停止相关自动运维操作            │
│    - 取消进行中的操作                │
│    - 暂停待执行的操作                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 告警通知                          │
│    - 通知相关人员                    │
│    - 记录熔断原因                    │
│    - 多渠道通知                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 状态保存                          │
│    - 保存当前状态                    │
│    - 保存熔断上下文                  │
│    - 保存现场证据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 等待恢复                          │
│    - 等待人工确认                    │
│    - 等待问题修复                    │
│    - 等待审批恢复                    │
└─────────────────────────────────────┘
```

### 7.4 熔断恢复

| 步骤 | 说明 | 审批要求 | 负责人 |
|------|------|----------|--------|
| 问题修复 | 修复导致熔断的问题 | - | 技术团队 |
| 恢复申请 | 申请恢复自动运维 | 按熔断级别 | 申请人 |
| 灰度恢复 | 先恢复部分操作 | 服务Owner | 运维团队 |

| 全量恢复 | 确认后全量恢复 | 按熔断级别 | 运维团队 |

## 八、Kill Switch

### 8.1 Kill Switch 定义

| 类型 | 代码 | 说明 | 触发方式 | 影响范围 |
|------|------|------|----------|----------|
| 紧急停止 | EMERGENCY | 立即停止所有自动运维 | 手动触发 | 全平台 |
| 服务停止 | SERVICE | 停止特定服务自动运维 | 手动触发 | 单服务 |
| 区域停止 | REGION | 停止特定区域自动运维 | 手动触发 | 单区域 |
| 租户停止 | TENANT | 停止特定租户自动运维 | 手动触发 | 单租户 |
| 操作停止 | OPERATION | 停止特定操作类型 | 手动触发 | 单操作类型 |

### 8.2 Kill Switch 触发

| 触发者 | 说明 | 权限要求 | 可触发类型 |
|--------|------|----------|------------|
| 值班人员 | 紧急情况触发 | 值班权限 | EMERGENCY, SERVICE |
| 服务Owner | 服务级触发 | 服务Owner权限 | SERVICE, OPERATION |
| 技术负责人 | 区域级触发 | 技术负责人权限 | REGION, SERVICE |
| 技术总监 | 全局触发 | 技术总监权限 | EMERGENCY, REGION |
| CTO | 最高级触发 | CTO权限 | 所有类型 |

### 8.3 Kill Switch 流程

```
Kill Switch 触发
    ↓
┌─────────────────────────────────────┐
│ 1. 权限验证                          │
│    - 验证触发者权限                  │
│    - 验证触发范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 立即停止                          │
│    - 停止所有相关操作                │
│    - 取消进行中操作                  │
│    - 阻止新操作启动                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 状态记录                          │
│    - 记录触发原因                    │
│    - 记录触发时间                    │
│    - 记录触发者                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 通知发布                          │
│    - 通知所有相关方                  │
│    - 发布停止公告                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 等待恢复                          │
│    - 等待问题解决                    │

└─────────────────────────────────────┘
```
