# RUNBOOK_EXECUTION_ENGINE.md - Runbook 自动执行规则

**版本: V27.0**

## 目的
定义 Runbook 自动执行规则，确保常见运维手册可从文档变成执行引擎，实现"Runbook 可执行、执行可审计、失败可回滚"。

## 适用范围
所有 Runbook 自动执行场景，包括：
- Runbook 注册
- 条件触发
- 执行权限
- 证据留存
- 回滚
- 人工接管

---

## 一、Runbook 定义

### 1.1 Runbook 结构

| 组成部分 | 说明 | 必需 | 类型 |
|----------|------|------|------|
| runbook_id | Runbook 唯一标识 | 是 | string |
| runbook_name | Runbook 名称 | 是 | string |
| description | Runbook 描述 | 是 | string |
| version | 版本号 | 是 | string |
| trigger_conditions | 触发条件 | 是 | array |
| steps | 执行步骤 | 是 | array |
| rollback_steps | 回滚步骤 | 是 | array |
| permissions | 执行权限 | 是 | object |
| timeout | 超时时间 | 是 | number |
| retry_policy | 重试策略 | 否 | object |
| notification | 通知配置 | 否 | object |

### 1.2 Runbook 类型

| 类型 | 代码 | 说明 | 自动执行 | 审批要求 |
|------|------|------|----------|----------|
| 自动 | AUTO | 可自动执行 | 是 | 无 |
| 半自动 | SEMI_AUTO | 需确认后执行 | 需确认 | 单次确认 |
| 手动 | MANUAL | 仅提供指导 | 否 | 每步确认 |
| 审批后自动 | APPROVED_AUTO | 审批后自动执行 | 审批后 | 事前审批 |

### 1.3 Runbook 示例

```json
{
  "runbook_id": "RB-001",
  "runbook_name": "服务自动重启",
  "description": "当服务异常时自动重启服务",
  "version": "1.0.0",
  "type": "AUTO",
  "trigger_conditions": [
    {
      "type": "alert",
      "condition": "service_down",
      "severity": "P1"
    }
  ],
  "steps": [
    {
      "step_id": "S1",
      "name": "检查服务状态",
      "action": "check_service_status",
      "params": {"service": "{{service_name}}"},
      "timeout": 30
    },
    {
      "step_id": "S2",
      "name": "重启服务",
      "action": "restart_service",
      "params": {"service": "{{service_name}}"},
      "timeout": 60
    },
    {
      "step_id": "S3",
      "name": "验证服务恢复",
      "action": "verify_service_health",
      "params": {"service": "{{service_name}}"},
      "timeout": 30
    }
  ],
  "rollback_steps": [
    {
      "step_id": "R1",
      "name": "回滚服务配置",
      "action": "rollback_config",
      "params": {"service": "{{service_name}}"},
      "timeout": 30
    }
  ],
  "permissions": {
    "execute": ["ops-team", "sre-team"],
    "manage": ["sre-lead"]
  },
  "timeout": 300,
  "retry_policy": {
    "max_retries": 2,
    "retry_interval": 10
  },
  "notification": {
    "on_start": ["ops-team"],
    "on_success": ["ops-team", "service-owner"],
    "on_failure": ["ops-team", "sre-lead"]
  }
}
```

## 二、Runbook 注册

### 2.1 注册流程

```
Runbook 提交
    ↓
┌─────────────────────────────────────┐
│ 1. 格式验证                          │
│    - 验证 Runbook 格式              │
│    - 验证步骤完整性                  │
│    - 验证参数有效性                  │
│    - 验证回滚步骤                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 安全审核                          │
│    - 审核执行权限                    │
│    - 审核操作风险                    │
│    - 审核影响范围                    │
│    - 审核回滚可行性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 测试验证                          │
│    - 测试环境验证                    │
│    - 验证执行效果                    │
│    - 验证回滚效果                    │
│    - 验证超时处理                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 审批发布                          │
│    - 提交审批                        │
│    - 审批人审批                      │
│    - 注册到 Runbook 库              │
│    - 发布可用                        │
└─────────────────────────────────────┘
```

### 2.2 注册要求

| 要求 | 说明 | 验证方式 |
|------|------|----------|
| 格式规范 | 符合 Runbook 格式规范 | 自动验证 |
| 权限明确 | 执行权限明确 | 人工审核 |
| 回滚完整 | 回滚步骤完整 | 自动验证 |
| 测试通过 | 测试验证通过 | 测试报告 |
| 文档完整 | 说明文档完整 | 人工审核 |
| 风险评估 | 风险评估完成 | 人工审核 |

## 三、条件触发

### 3.1 触发类型

| 类型 | 代码 | 说明 | 示例 | 响应时间 |
|------|------|------|------|----------|
| 告警触发 | ALERT | 告警触发执行 | CPU > 90% | < 1分钟 |
| 事件触发 | EVENT | 事件触发执行 | 服务重启 | < 1分钟 |
| 定时触发 | SCHEDULE | 定时触发执行 | 日志清理 | 准时 |
| 手动触发 | MANUAL | 手动触发执行 | 人工启动 | 即时 |
| Webhook触发 | WEBHOOK | 外部系统触发 | CI/CD触发 | < 1分钟 |
| API触发 | API | API调用触发 | 自动化系统 | < 1分钟 |

### 3.2 触发条件

| 条件类型 | 说明 | 示例 |
|----------|------|------|
| 指标条件 | 监控指标触发 | cpu_usage > 90% |
| 事件条件 | 事件类型触发 | service_crashed |
| 时间条件 | 定时触发 | cron: 0 2 * * * |
| 组合条件 | 多条件组合 | cpu > 90% AND memory > 80% |
| 表达式条件 | 复杂表达式 | (cpu > 90% OR memory > 90%) AND duration > 5m |

### 3.3 触发配置

```json
{
  "trigger_config": {
    "trigger_id": "TRG-001",
    "runbook_id": "RB-001",
    "type": "ALERT",
    "conditions": [
      {
        "metric": "service_status",
        "operator": "equals",
        "value": "down",
        "duration": "1m"
      }
    ],
    "filters": {
      "service": ["api-gateway", "auth-service"],
      "severity": ["P0", "P1"]
    },
    "throttle": {
      "cooldown": "5m",
      "max_triggers": 3,
      "window": "1h"
    }
  }
}
```

### 3.4 触发流程

```
触发条件满足
    ↓
┌─────────────────────────────────────┐
│ 1. 条件验证                          │
│    - 验证触发条件                    │
│    - 确认触发有效性                  │
│    - 检查触发频率限制                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. Runbook 匹配                      │
│    - 匹配对应 Runbook               │
│    - 验证执行条件                    │
│    - 检查 Runbook 状态              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行准备                          │
│    - 准备执行环境                    │
│    - 验证执行权限                    │
│    - 准备执行参数                    │
│    - 发送开始通知                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 执行启动                          │
│    - 启动 Runbook 执行              │
│    - 记录执行日志                    │
│    - 监控执行状态                    │
└─────────────────────────────────────┘
```

## 四、执行权限

### 4.1 权限分级

| 级别 | 代码 | 说明 | 允许操作 |
|------|------|------|----------|
| L1-只读 | READ | 只读权限 | 查询、诊断 |
| L2-执行 | EXECUTE | 执行权限 | 执行 Runbook |
| L3-管理 | MANAGE | 管理权限 | 注册、修改 Runbook |
| L4-管理员 | ADMIN | 管理员权限 | 所有操作 |

### 4.2 权限控制

| 控制项 | 说明 | 验证方式 |
|--------|------|----------|
| Runbook 权限 | Runbook 执行权限 | RBAC |
| 操作权限 | 具体操作权限 | 操作白名单 |
| 范围权限 | 执行范围权限 | 资源标签 |
| 时间权限 | 执行时间权限 | 时间窗口 |

### 4.3 权限配置

```json
{
  "permission_config": {
    "runbook_id": "RB-001",
    "permissions": {
      "execute": {
        "users": ["user1@example.com"],
        "groups": ["ops-team", "sre-team"],
        "roles": ["operator", "sre"]
      },
      "manage": {
        "users": ["sre-lead@example.com"],
        "groups": ["sre-leads"],
        "roles": ["admin"]
      }
    },
    "scope": {
      "services": ["api-gateway", "auth-service"],
      "regions": ["cn-north", "cn-south"],
      "environments": ["production", "staging"]
    }
  }
}
```

### 4.4 权限验证

| 验证项 | 说明 | 失败处理 |
|--------|------|----------|
| 用户权限 | 验证用户权限 | 拒绝执行 |
| Runbook 权限 | 验证 Runbook 权限 | 拒绝执行 |
| 操作权限 | 验证操作权限 | 拒绝执行 |
| 范围权限 | 验证范围权限 | 拒绝执行 |
| 时间权限 | 验证时间权限 | 延迟执行 |

## 六、回滚

### 6.1 回滚触发

| 触发条件 | 说明 | 自动触发 |
|----------|------|----------|
| 执行失败 | Runbook 执行失败 | 是 |
| 结果异常 | 执行结果异常 | 是 |
| 验证失败 | 结果验证失败 | 是 |
| 人工请求 | 人工请求回滚 | 否 |
| 超时 | 执行超时 | 是 |
| 依赖失败 | 依赖步骤失败 | 是 |

### 6.2 回滚策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 自动回滚 | 自动执行回滚 | 低风险操作 |
| 确认回滚 | 确认后执行回滚 | 中风险操作 |
| 人工回滚 | 人工执行回滚 | 高风险操作 |
| 部分回滚 | 部分步骤回滚 | 可恢复场景 |

### 6.3 回滚流程

```
回滚触发
    ↓
┌─────────────────────────────────────┐
│ 1. 回滚决策                          │
│    - 确认回滚必要性                  │
│    - 选择回滚方案                    │
│    - 评估回滚风险                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 回滚准备                          │
│    - 准备回滚环境                    │
│    - 验证回滚条件                    │
│    - 备份当前状态                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 回滚执行                          │
│    - 执行回滚步骤                    │
│    - 记录回滚日志                    │
│    - 监控回滚状态                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 回滚验证                          │
│    - 验证回滚结果                    │
│    - 确认系统恢复                    │
│    - 验证数据一致性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 回滚完成                          │
│    - 生成回滚报告                    │
│    - 通知相关人员                    │
│    - 更新执行状态                    │
└─────────────────────────────────────┘
```

### 6.4 回滚验证

| 验证项 | 说明 | 验证方式 |
|--------|------|----------|
| 状态恢复 | 状态恢复验证 | 状态检查 |
| 功能正常 | 功能正常验证 | 功能测试 |
| 数据一致 | 数据一致性验证 | 数据校验 |
| 配置正确 | 配置正确性验证 | 配置比对 |
