# LATENCY_POLICY.md - 时延策略

## 目的
定义时延目标与时延预算，确保时延管理可控。

## 适用范围
所有任务执行的时延控制。

## 时延目标

### 按任务类型
| 任务类型 | 首答目标 | 完成目标 | 说明 |
|----------|----------|----------|------|
| 简单问答 | 1s | 3s | 速度优先 |
| 信息搜索 | 2s | 10s | 平衡 |
| 文档编辑 | 3s | 30s | 质量优先 |
| 复杂分析 | 5s | 60s | 质量优先 |
| 方案规划 | 5s | 120s | 质量优先 |

### 按优先级
| 优先级 | 首答目标 | 说明 |
|--------|----------|------|
| P0 | 1s | 最高优先 |
| P1 | 2s | 高优先 |
| P2 | 5s | 标准优先 |
| P3 | 10s | 低优先 |

## 时延预算分配

### 单任务时延预算
| 阶段 | 预算占比 | 说明 |
|------|----------|------|
| 输入分析 | 5% | 快速分析 |
| 任务分类 | 5% | 快速分类 |
| 检索执行 | 40% | 主要耗时 |
| 结果处理 | 20% | 整合处理 |
| 输出生成 | 30% | 格式化输出 |

### 多步骤时延预算
| 阶段 | 预算占比 | 说明 |
|------|----------|------|
| 规划阶段 | 10% | 任务规划 |
| 执行阶段 | 60% | 主要执行 |
| 校验阶段 | 20% | 结果校验 |
| 整合阶段 | 10% | 结果整合 |

## 速度与准确性权衡

### 速度优先场景
| 场景 | 说明 |
|------|------|
| 用户等待中 | 用户正在等待响应 |
| 简单问题 | 简单问题快速回答 |
| 交互对话 | 实时交互场景 |
| 低风险任务 | 低风险可接受快速响应 |

**策略**:
- 使用快速模型
- 减少检索深度
- 简化校验流程
- 缓存优先

### 准确性优先场景
| 场景 | 说明 |
|------|------|
| 高风险任务 | 需要高准确性 |
| 事实密集 | 大量事实陈述 |
| 用户要求 | 用户要求准确 |
| 关键决策 | 影响重要决策 |

**策略**:
- 使用深度模型
- 完整检索流程
- 多重校验
- 来源验证

## 阶段性结果输出

### 何时先给阶段性结果
| 条件 | 说明 |
|------|------|
| 任务时长 > 30s | 长任务 |
| 用户请求进度 | 用户想知道进度 |
| 关键阶段完成 | 重要阶段完成 |
| 等待用户决策 | 需要用户确认 |

### 阶段性结果格式
```
[进度更新]
当前阶段: {阶段名称}
完成进度: {百分比}%
已完成: {已完成内容}
下一步: {下一步操作}
预计剩余: {预计时间}
```

### 何时必须等校验完成
| 条件 | 说明 |
|------|------|
| 高风险输出 | 涉及安全风险 |
| 事实密集 | 大量事实陈述 |
| 用户要求准确 | 用户明确要求 |
| 来源冲突 | 存在信息冲突 |

## 时延监控

### 监控指标
| 指标 | 说明 |
|------|------|
| 首答时延 | 首次响应时间 |
| 完成时延 | 任务完成时间 |
| 阶段时延 | 各阶段耗时 |
| 等待时间 | 用户等待时间 |

### 告警阈值
| 指标 | 警告阈值 | 错误阈值 |
|------|----------|----------|
| 首答时延 | > 3s | > 5s |
| 完成时延 | > 30s | > 60s |
| 阶段时延 | > 预算150% | > 预算200% |

## 时延优化策略

### 检测到时延过高
| 策略 | 说明 |
|------|------|
| 模型降级 | 使用更快模型 |
| 检索简化 | 减少检索深度 |
| 并行执行 | 并行化处理 |
| 缓存利用 | 利用缓存 |

### 检测到时延过低
| 策略 | 说明 |
|------|------|
| 质量提升 | 增加校验 |
| 来源验证 | 增加验证 |
| 深度分析 | 更深入分析 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 时延超限 | 返回部分结果 |
| 阶段超时 | 跳过或降级 |
| 用户取消 | 终止执行 |

## 维护方式
- 新增任务类型: 添加到时延目标表
- 调整预算分配: 更新预算分配表
- 新增优化策略: 更新优化策略表
