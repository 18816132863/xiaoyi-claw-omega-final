# 性能与架构优化方案 V4.2.0

## 问题诊断

### 当前瓶颈
| 问题 | 数据 | 影响 |
|------|------|------|
| 文件过多 | 9018 个 | Token 消耗大 |
| 目录过大 | 126MB | 加载慢 |
| 无分层 | 全部平等 | 效率低 |
| 无缓存 | 每次重载 | 重复计算 |

## 优化方案

### 1. 分层懒加载
```
P0 核心层 (立即加载)
├── 9 个核心文件
├── < 50KB
└── 3000 tokens

P1 常用层 (按需加载)
├── 路由、网关、搜索
├── < 500KB
└── 2000 tokens

P2 扩展层 (延迟加载)
├── 技能 (单个加载)
├── < 2MB
└── 1500 tokens

P3 归档层 (不加载)
├── repo/, reports/
├── *.pdf, *.log
└── 0 tokens
```

### 2. 技能索引优化
```
倒排索引:
  触发词 → 技能列表
  分类 → 技能列表
  层级 → 技能列表

热点缓存:
  Top 10 常用技能预加载
  LRU 淘汰策略
  5MB 内存限制
```

### 3. 多级缓存
```
L1 内存缓存
├── 热点数据
├── < 1ms 命中
└── 100 项 / 5分钟

L2 文件缓存
├── 常用数据
├── < 10ms 命中
└── 10MB / 1小时

L3 压缩缓存
├── 历史数据
├── < 100ms 命中
└── 50MB / 1天
```

### 4. Token 预算
```
L1 核心层: 3000 tokens
L2 记忆层: 2000 tokens
L3 编排层: 1500 tokens
L4 执行层: 1500 tokens
L5 治理层: 800 tokens
L6 基础层: 700 tokens
─────────────────────
总计: ~9500 tokens
```

### 5. 增量更新
```
变更检测: mtime 对比
索引更新: 仅更新变更部分
同步机制: 差异同步
```

## 预期效果

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 启动 Token | ~15000 | ~3000 | 80% ↓ |
| 首次加载 | ~5s | ~0.5s | 90% ↓ |
| 技能查找 | ~500ms | ~10ms | 98% ↓ |
| 内存占用 | ~50MB | ~5MB | 90% ↓ |
