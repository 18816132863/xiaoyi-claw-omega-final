
## 📊 详细关系说明

### 1. OpenClaw 是什么？

**OpenClaw** 是一个 **AI Agent 运行平台**，提供：

| 功能 | 说明 |
|------|------|
| 运行时环境 | Node.js、Python 执行环境 |
| 工具链 | 文件操作、浏览器控制、消息发送等 |
| 会话管理 | 会话持久化、历史记录、上下文管理 |
| 技能系统 | 技能注册、路由、执行 |
| 通道集成 | Telegram、Discord、Signal、小艺等 |
| 记忆系统 | 向量存储、知识库、长期记忆 |

**核心特点**:
- 平台级基础设施
- 可运行多个不同的 AI Agent
- 提供统一的工具接口
- 支持多通道接入

---

### 2. 鸽子王架构是什么？

**鸽子王架构** 是运行在 OpenClaw 平台上的 **具体 AI Agent 实现**，包括：

| 层级 | 名称 | 职责 |
|------|------|------|
| L1 | Core | 核心认知、身份、规则、标准 |
| L2 | Memory Context | 记忆上下文、知识库、向量存储 |
| L3 | Orchestration | 任务编排、工作流、路由 |
| L4 | Execution | 能力执行、技能网关 |
| L5 | Governance | 稳定治理、安全审计 |
| L6 | Infrastructure | 基础设施、工具链 |

**核心特点**:
- 具体的 AI Agent 实现
- 定义了身份、性格、技能
- 实现了六层架构
- 包含 275 个技能

---

## 📝 实际运行流程

### 用户发送消息流程

```
1. 用户在小艺发送消息
   ↓
2. OpenClaw 接收消息 (xiaoyi-channel)
   ↓
3. OpenClaw 加载鸽子王架构
   ├── 读取 IDENTITY.md (身份)
   ├── 读取 SOUL.md (性格)
   ├── 读取 USER.md (用户信息)
   └── 读取 MEMORY.md (记忆)
   ↓
4. 鸽子王架构处理消息
   ├── L3 编排任务
   ├── L4 执行技能
   └── L5 治理检查
   ↓
5. 鸽子王架构返回结果
   ↓
6. OpenClaw 发送回复 (message tool)
   ↓
7. 用户收到回复
```
