## 二、体量优化方案

### 方案 A：清理历史备份（预计减少 497M）

**操作**：
```bash
# 保留最近 3 个备份，删除其余
cd archive/backups/
ls -t *.tar.gz | tail -n +4 | xargs rm -f
```

**预期效果**：从 497M 减少到 ~80M

### 方案 B：精简 Python 依赖（预计减少 100M）

**问题**：`repo/lib/python3.12/` 包含完整的 pandas、playwright 等大型库

**方案**：
1. 移除不常用的大型依赖
2. 使用系统 Python 库替代
3. 按需安装依赖

**操作**：
```bash
# 移除大型库，按需安装
rm -rf repo/lib/python3.12/site-packages/pandas
rm -rf repo/lib/python3.12/site-packages/playwright
```

### 方案 C：压缩记忆索引（预计减少 30M）

**问题**：`memory_context/index/` 的 JSON 文件未压缩

**方案**：
1. 使用 gzip 压缩索引文件
2. 运行时解压加载
3. 或使用二进制格式（pickle/msgpack）

**操作**：
```bash
gzip memory_context/index/*.json
```

### 方案 D：精简技能目录（预计减少 20M）

**问题**：276 个技能，部分包含大型二进制文件

**方案**：
1. 移除技能中的二进制文件
2. 技能按需下载资源
3. 清理重复技能

**操作**：
```bash
# 查找技能中的大文件
find skills/ -type f -size +500k -exec ls -lh {} \;
```

## 三、性能优化方案

### 问题 1：规则引擎启动慢

**原因**：每次执行都要加载所有规则和例外

**方案**：
1. 规则缓存：首次加载后缓存到内存
2. 增量检查：只检查变更的规则
3. 并行执行：多规则并行检查

### 问题 2：记忆索引加载慢

**原因**：`fts_index.json` (18M) 和 `keyword_index.json` (26M) 加载慢

**方案**：
1. 懒加载：按需加载索引
2. 分片索引：按时间/类型分片
3. 使用 SQLite：将 JSON 转为 SQLite 数据库

### 问题 3：技能扫描慢

**原因**：276 个技能，每个都要检查 SKILL.md

**方案**：
1. 技能注册表缓存
2. 增量更新：只扫描变更的技能
3. 技能索引：预构建技能索引
