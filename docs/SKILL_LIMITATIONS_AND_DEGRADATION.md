# 技能限制与降级

## 默认模式限制

### 存储限制

| 限制 | 说明 | 影响 |
|------|------|------|
| 单文件数据库 | SQLite 单文件 | 不支持分布式 |
| 无复制 | 无主从复制 | 无高可用 |
| 本地存储 | 数据在本地 | 无法跨机器共享 |

### 执行限制

| 限制 | 说明 | 影响 |
|------|------|------|
| 单进程 | 单进程执行 | 并发受限 |
| 无队列 | 无消息队列 | 无异步解耦 |
| 内存缓存 | 无 Redis | 重启丢失缓存 |

### 调度限制

| 限制 | 说明 | 影响 |
|------|------|------|
| 本地调度 | 无分布式调度 | 单机调度 |
| 精度有限 | 分钟级调度 | 不支持秒级 |
| 无持久化队列 | 重启丢失待执行 | 可能漏任务 |

## 降级场景

### 场景1: 平台不可用

**触发条件**: 小艺/鸿蒙平台API不可用

**降级行为**:
- 使用本地调度替代平台调度
- 使用本地记录替代平台消息
- 记录降级事件

**用户提示**:
```
"已降级为本地模式，部分功能受限"
```

### 场景2: 数据库不可用

**触发条件**: SQLite 文件损坏或不可写

**降级行为**:
- 使用内存存储
- 记录警告日志
- 提示用户修复

**用户提示**:
```
"数据库不可用，使用临时存储，重启后数据将丢失"
```

### 场景3: 磁盘空间不足

**触发条件**: 磁盘空间低于阈值

**降级行为**:
- 停止历史记录
- 仅保留活跃任务
- 触发清理策略

**用户提示**:
```
"磁盘空间不足，已停止历史记录"
```

## 错误解释

当发生错误时，系统提供人类可读的解释：

```python
from diagnostics.error_explainer import ErrorExplainer

explanation = ErrorExplainer.explain("E2004")
# 返回:
{
    "title": "任务状态无效",
    "explanation": "当前任务状态不允许执行此操作",
    "common_causes": [...],
    "solutions": [...]
}
```

## 最佳实践

1. **定期备份** - 导出任务历史
2. **监控磁盘** - 保持足够空间
3. **检查日志** - 关注降级事件
4. **适时升级** - 需要时切换到增强模式
