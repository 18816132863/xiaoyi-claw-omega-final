# 技能用户结果说明

## 版本
- V8.3.0 路径统一版
- 更新日期: 2026-04-24

## 用户可见结果类型

### 1. 立即完成 (Immediate Completion)
**条件**: 平台已连接，操作同步完成

**用户看到**:
```
✅ 已完成
```

**内部状态**: `succeeded`

---

### 2. 已提交等待确认 (Submitted Pending Confirmation)
**条件**: 操作已提交，等待平台确认

**用户看到**:
```
⏳ 已提交，等待处理...
```

**内部状态**: `delivery_pending`

---

### 3. 已排队 (Queued)
**条件**: 任务已入队，等待执行

**用户看到**:
```
📋 已排队 (位置: #3)
```

**内部状态**: `queued`

---

### 4. 执行中 (Running)
**条件**: 任务正在执行

**用户看到**:
```
🔄 正在执行...
```

**内部状态**: `running`

---

### 5. 需要重试 (Retry Needed)
**条件**: 执行失败，等待重试

**用户看到**:
```
⚠️ 执行遇到问题，正在重试 (2/3)...
```

**内部状态**: `waiting_retry`

---

### 6. 需要人工介入 (Human Required)
**条件**: 需要用户确认或操作

**用户看到**:
```
🙋 需要您的确认
[确认] [取消]
```

**内部状态**: `waiting_human`

---

### 7. 失败 (Failed)
**条件**: 执行失败，无法恢复

**用户看到**:
```
❌ 执行失败
原因: [错误信息]
```

**内部状态**: `failed`

---

## 状态转换用户通知

| 从状态 | 到状态 | 用户通知 |
|--------|--------|----------|
| queued | running | "开始执行..." |
| running | delivery_pending | "已提交，等待确认" |
| delivery_pending | succeeded | "✅ 完成" |
| delivery_pending | failed | "❌ 送达失败" |
| running | waiting_retry | "遇到问题，重试中..." |
| waiting_retry | succeeded | "✅ 重试成功" |

## 平台能力对用户的影响

### 平台已连接 (Connected)
- 用户看到实时状态更新
- 可以获得最终确认
- 状态准确可靠

### 平台仅探测 (Probe Only)
- 用户看到"已提交"而非"已完成"
- 无法获得最终确认
- 状态可能不准确

### 降级模式 (Fallback)
- 用户看到"已加入队列"
- 需要外部系统处理
- 状态依赖外部系统

## 用户结果解释模板

### 定时消息
```
📅 定时消息已创建
发送时间: 2026-04-24 15:00
状态: 已排队

💡 提示: 消息将在指定时间自动发送
```

### 即时消息
```
💬 消息已发送
状态: ✅ 已送达

💡 提示: 消息已成功送达目标
```

### 消息（等待确认）
```
💬 消息已提交
状态: ⏳ 等待送达确认

💡 提示: 消息已加入发送队列，等待网关处理
```

|----------|----------|

| `DELIVERY_TIMEOUT` | "处理超时，请稍后重试" |
| `RETRY_EXHAUSTED` | "多次尝试后仍失败，请联系支持" |
