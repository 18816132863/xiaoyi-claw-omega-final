# 长期治理规则

## 一、新技能接入审批流程

### 1.1 审批流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     新技能接入审批流程                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 提交接入设计卡                                                      │
│     ↓                                                                   │
│  2. 层级归属审批                                                        │
│     ↓                                                                   │
│  3. 风险评估审批                                                        │
│     ↓                                                                   │
│  4. 回滚方案审批                                                        │
│     ↓                                                                   │
│  5. 开发实现                                                            │
│     ↓                                                                   │
│  6. 自动检查                                                            │
│     ↓                                                                   │
│  7. 灰度验证                                                            │
│     ↓                                                                   │
│  8. 正式上线                                                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 审批要求

| 审批项 | 审批人 | 要求 |
|-------|-------|------|
| 层级归属 | 系统架构师 | 必须符合六层架构定义 |
| 接入设计 | 系统架构师 | 必须按统一模板设计 |
| 风险评估 | 安全工程师 | 必须识别所有风险点 |
| 回滚方案 | 运维工程师 | 必须可独立回滚 |

## 二、技能废弃/下线流程

### 2.1 废弃条件

| 条件 | 说明 |
|-----|------|
| 长期无调用 | 90天内无调用记录 |
| 功能重复 | 与其他技能功能重复 |
| 技术过时 | 使用过时的技术或API |
| 安全风险 | 存在无法修复的安全风险 |

### 2.2 下线流程

```
1. 标记为 deprecated
   ↓
2. 通知所有使用方
   ↓
3. 等待30天迁移期
   ↓
4. 标记为 offline
   ↓
5. 归档到 _archive/deprecated/
   ↓
6. 从注册表中删除
```

### 2.3 下线检查

| 检查项 | 要求 |
|-------|------|
| 无调用记录 | 确认无任何调用 |
| 无依赖关系 | 确认无其他技能依赖 |
| 数据已迁移 | 确认数据已迁移或备份 |
| 文档已更新 | 确认文档已更新 |

## 三、注册表更新规则

### 3.1 更新时机

| 时机 | 操作 |
|-----|------|
| 新增技能 | 添加新记录 |
| 技能升级 | 更新 version 字段 |
| 技能废弃 | 更新 status 为 deprecated |
| 技能下线 | 删除记录 |

### 3.2 更新规则

| 规则 | 说明 |
|-----|------|
| 唯一性 | skill_id 必须唯一 |
| 完整性 | 必填字段不能为空 |
| 版本控制 | 每次更新必须更新 version |
| 时间戳 | 每次更新必须更新 updated_at |

### 3.3 更新命令

```bash
# 查看注册表
cat infrastructure/inventory/skill_registry.json

# 编辑注册表
vim infrastructure/inventory/skill_registry.json

# 验证注册表
python3 infrastructure/inventory/skill_access_checker.py
```

## 四、挂层表更新规则

### 4.1 更新时机

| 时机 | 操作 |
|-----|------|
| 新增技能 | 添加到对应层级 |
| 技能迁移 | 更新层级归属 |

| 路径正确 | 必须与实际路径一致 |
| 统计准确 | 技能数必须准确 |

## 五、冗余模块定期清理机制

### 5.1 清理周期

| 清理项 | 周期 | 说明 |
|-------|------|------|
| 临时文件 | 每天 | 删除临时文件 |
| 日志文件 | 每周 | 压缩并归档日志 |
| 缓存文件 | 每周 | 清理过期缓存 |
| 归档文件 | 每月 | 删除超过90天的归档 |
| 废弃技能 | 每月 | 归档废弃技能 |

### 5.2 清理命令

```bash
# 清理临时文件
find . -name "*.tmp" -delete

# 清理日志文件
find logs/ -name "*.log" -mtime +7 -exec gzip {} \;

# 清理缓存文件
find . -name "__pycache__" -exec rm -rf {} +

# 清理归档文件
find _archive/ -mtime +90 -delete
```

## 六、架构巡检周期

### 6.1 巡检内容

| 巡检项 | 周期 | 检查内容 |
|-------|------|---------|
| 目录结构 | 每周 | 检查是否符合六层架构 |
| 注册表 | 每周 | 检查是否完整准确 |
| 越层调用 | 每周 | 检查是否存在越层调用 |
| 游离模块 | 每周 | 检查是否存在游离模块 |
| 配置源 | 每月 | 检查是否配置源唯一 |
| 依赖关系 | 每月 | 检查依赖关系是否合理 |

### 6.2 巡检命令

```bash
# 运行架构检查
python3 infrastructure/inventory/skill_access_checker.py

# 运行违规测试
python3 infrastructure/inventory/violation_test_suite.py

# 检查目录结构
ls -la | grep -E "^d" | grep -v "^\."
```

## 七、禁止事项

### 7.1 禁止私自新增

| 禁止项 | 说明 |
|-------|------|
| 入口 | 禁止私自新增技能入口 |
| 注册点 | 禁止私自新增注册点 |
| 配置源 | 禁止私自新增配置源 |
| 层级 | 禁止私自新增层级 |

### 7.2 禁止越层调用

| 禁止项 | 说明 |
|-------|------|
| 向上调用 | 禁止下层调用上层 |
| 跨层调用 | 禁止跳过中间层 |
| 循环调用 | 禁止循环依赖 |

### 7.3 禁止绕过检查

| 禁止项 | 说明 |
|-------|------|
| 绕过注册 | 禁止未注册直接接入 |
| 绕过检查 | 禁止跳过自动检查 |
| 绕过审批 | 禁止跳过审批流程 |

## 八、违规处理

### 8.1 违规等级

| 等级 | 说明 | 处理方式 |
|-----|------|---------|
| 低 | 警告级别 | 通知整改 |
| 中 | 影响架构 | 强制整改 |
| 高 | 破坏架构 | 立即回滚 |

### 8.2 处理流程

```
1. 发现违规
   ↓
2. 评估等级
   ↓
3. 通知责任人
   ↓
4. 限期整改
   ↓
5. 验证整改
   ↓
6. 记录归档
```
