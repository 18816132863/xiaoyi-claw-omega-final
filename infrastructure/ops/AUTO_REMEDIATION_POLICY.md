# AUTO_REMEDIATION_POLICY.md - 自动修复规则

**版本: V27.0**

## 目的
定义自动修复规则，确保平台开始具备受控自动运维能力，实现"故障可自动修复、修复有边界、执行可审计"。

## 适用范围
所有自动修复场景，包括：
- 可自动修复的故障
- 需审批后修复的故障
- 只能建议不能自动执行的故障
- 修复失败后的升级路径

---

## 一、自动修复原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| 安全优先 | 自动修复不能引入新风险 | 风险评估 + 前置检查 |
| 边界明确 | 明确可自动修复的范围 | 白名单 + 黑名单 |
| 审计完整 | 所有自动修复有审计记录 | 全量审计日志 |
| 可回滚 | 自动修复可回滚 | 回滚步骤定义 |
| 可熔断 | 异常时可熔断停止 | 熔断机制 |
| 可追溯 | 修复过程可追溯 | 执行链路记录 |
| 可预测 | 修复结果可预测 | 预检查 + 模拟 |

## 二、故障分类与修复策略

### 2.1 故障分类

| 分类 | 代码 | 说明 | 修复策略 | 审批要求 |
|------|------|------|----------|----------|
| 自动修复 | AUTO | 可安全自动修复 | 自动执行 | 无 |
| 审批修复 | APPROVAL | 需审批后修复 | 审批后执行 | 事前审批 |
| 建议修复 | SUGGEST | 只能建议 | 发送建议 | 无 |
| 人工修复 | MANUAL | 必须人工处理 | 告警通知 | 人工处理 |
| 禁止修复 | FORBIDDEN | 禁止自动修复 | 仅告警 | 禁止 |

### 2.2 故障类型与策略

| 故障类型 | 修复策略 | 风险等级 | 说明 |
|----------|----------|----------|------|
| 服务无响应 | AUTO | 低 | 自动重启服务 |
| 内存泄漏 | AUTO | 低 | 自动重启进程 |
| 连接池耗尽 | AUTO | 低 | 自动重置连接池 |
| 进程僵死 | AUTO | 低 | 终止并重启进程 |
| 队列积压 | AUTO | 低 | 扩展消费者 |
| 磁盘空间不足 | APPROVAL | 中 | 需审批后清理 |
| 配置错误 | APPROVAL | 中 | 需审批后修复 |
| 证书过期 | APPROVAL | 中 | 需审批后续期 |
| 依赖服务异常 | APPROVAL | 中 | 需审批后切换 |
| 数据损坏 | SUGGEST | 高 | 只能建议修复方案 |
| 性能异常 | SUGGEST | 中 | 只能建议优化方案 |
| 安全风险 | SUGGEST | 高 | 只能建议修复方案 |
| 容量预警 | SUGGEST | 中 | 只能建议扩展方案 |
| 安全事件 | MANUAL | 高 | 必须人工处理 |
| 数据丢失 | MANUAL | 高 | 必须人工处理 |
| 核心服务故障 | MANUAL | 高 | 必须人工处理 |
| 数据库故障 | FORBIDDEN | 高 | 禁止自动修复 |

## 八、修复边界

### 8.1 服务边界

| 边界 | 说明 | 检查方式 |
|------|------|----------|
| 服务范围 | 仅限已注册服务 | 服务白名单 |
| 服务等级 | 按服务等级限制修复动作 | SLA配置 |
| 服务依赖 | 考虑服务依赖关系 | 依赖图谱 |

## 九、熔断机制

### 9.1 熔断触发条件

| 条件 | 说明 | 阈值 |
|------|------|------|
| 修复失败率高 | 失败率过高 | > 30% |
| 影响范围扩大 | 修复导致新故障 | 检测到新故障 |
| 连续失败 | 连续失败次数过多 | > 3 次 |
| 人工干预 | 人工请求停止 | 人工触发 |
| 系统异常 | 系统检测到异常 | 异常检测 |
| 外部事件 | 外部重大事件 | 事件触发 |

### 9.2 熔断动作

| 动作 | 说明 | 执行顺序 |
|------|------|----------|
| 停止修复 | 停止所有自动修复 | 1 |
| 告警通知 | 通知相关人员 | 2 |
| 人工接管 | 转人工处理 | 3 |
| 审计记录 | 记录熔断原因 | 4 |
| 状态更新 | 更新系统状态 | 5 |

### 9.3 熔断恢复

| 步骤 | 说明 | 负责人 |
|------|------|--------|
| 问题修复 | 修复导致熔断的问题 | 技术团队 |
| 审批确认 | 负责人确认恢复 | 技术负责人 |
| 灰度恢复 | 先恢复部分修复 | 运维团队 |
| 监控验证 | 验证修复正常 | 监控团队 |
| 全量恢复 | 确认后全量恢复 | 运维团队 |
