# MAINTENANCE_WINDOW_POLICY.md - 维护窗口规则

**版本: V27.0**

## 目的
定义维护窗口规则，确保平台维护活动被正式治理，不再随意进行，实现"维护有计划、通知有标准、影响有控制"。

## 适用范围
所有维护窗口场景，包括：
- 计划维护
- 区域维护
- 租户维护
- 紧急维护
- 维护期间能力限制

---

## 一、维护类型

| 类型 | 代码 | 说明 | 审批要求 | 提前通知 |
|------|------|------|----------|----------|
| 计划维护 | PLANNED | 计划内维护 | 技术负责人 | 7天 |
| 区域维护 | REGION | 区域维护 | 技术负责人 | 3天 |
| 租户维护 | TENANT | 租户维护 | 服务Owner | 7天 |
| 紧急维护 | EMERGENCY | 紧急维护 | 事后补审批 | 立即 |
| 滚动维护 | ROLLING | 滚动维护 | 运维负责人 | 3天 |
| 升级维护 | UPGRADE | 版本升级维护 | 技术总监 | 14天 |

### 1.1 维护级别定义

| 级别 | 代码 | 影响范围 | 审批级别 |
|------|------|----------|----------|
| L1-轻度 | LIGHT | 单服务节点 | 服务Owner |
| L2-中度 | MEDIUM | 多服务/单区域 | 技术负责人 |
| L3-重度 | HEAVY | 跨区域/多服务 | 技术总监 |
| L4-全局 | GLOBAL | 全局影响 | CTO |

### 1.2 维护优先级

| 优先级 | 说明 | 响应时间 |
|--------|------|----------|
| P0-紧急 | 立即执行 | < 30分钟 |
| P1-高优 | 尽快执行 | < 4小时 |
| P2-标准 | 计划执行 | 下个窗口 |
| P3-低优 | 灵活执行 | 任意窗口 |

## 二、计划维护

### 2.1 维护窗口

| 窗口类型 | 时间段 | 说明 | 适用维护 |
|----------|--------|------|----------|
| 常规窗口 | 周二/周四 02:00-06:00 | 常规维护 | L1/L2维护 |
| 扩展窗口 | 周六 00:00-08:00 | 大规模维护 | L3维护 |
| 禁止窗口 | 月初/月末 | 禁止维护 | 无 |
| 节假日窗口 | 节假日除外 | 节假日禁止 | 无 |
| 特殊窗口 | 协商确定 | 特殊维护 | L4维护 |

### 2.2 维护申请

| 申请项 | 说明 | 必填 |
|--------|------|------|
| 维护内容 | 维护内容描述 | 是 |
| 影响范围 | 影响范围评估 | 是 |
| 维护时长 | 预计维护时长 | 是 |
| 回滚方案 | 回滚方案说明 | 是 |
| 验证方案 | 验证方案说明 | 是 |
| 监控方案 | 监控方案说明 | 是 |
| 通知范围 | 通知范围说明 | 是 |
| 应急预案 | 应急预案说明 | L3+必填 |

### 2.3 维护申请表单

```json
{
  "maintenance_request": {
    "request_id": "MNT-2026-001",
    "type": "PLANNED",
    "level": "L2",
    "title": "API网关升级维护",
    "description": "升级API网关至v2.0版本",
    "window": {
      "start": "2026-04-15T02:00:00Z",
      "end": "2026-04-15T06:00:00Z"
    },
    "impact": {
      "services": ["api-gateway", "auth-service"],
      "regions": ["cn-north"],
      "tenants": ["all"]
    },
    "rollback_plan": "回滚至v1.9.5",
    "verification_plan": "API测试+性能测试",
    "monitoring_plan": "监控告警+日志跟踪",
    "notify_scope": ["all-tenants", "ops-team"],
    "emergency_plan": "快速回滚+流量切换"
  }
}
```

## 四、租户维护

### 4.1 租户维护窗口

| 租户类型 | 维护窗口 | 说明 | 协商要求 |
|----------|----------|------|----------|
| 企业租户 | 协商确定 | 与租户协商 | 提前14天 |
| 标准租户 | 周二/周四 | 标准窗口 | 提前7天 |
| 试用租户 | 随时 | 灵活安排 | 提前3天 |
| VIP租户 | 协商确定 | 专属窗口 | 提前30天 |
| 内部租户 | 周末 | 内部窗口 | 提前7天 |

### 4.2 租户维护通知

| 通知项 | 说明 | 时间要求 |
|--------|------|----------|
| 提前通知 | 提前通知租户 | 7天前 |
| 确认回复 | 确认租户收到 | 3天前 |
| 影响说明 | 说明维护影响 | 通知时 |
| 恢复通知 | 维护恢复通知 | 完成后 |
| 报告提供 | 提供维护报告 | 完成后24h |

### 4.3 租户维护模板

```

维护时间：{start_time} - {end_time}
维护时长：{duration}
影响范围：{impact_scope}
影响内容：{impact_detail}

维护期间影响：
- {impact_1}
- {impact_2}

建议您在维护前做好以下准备：
- {suggestion_1}
- {suggestion_2}

如有疑问，请联系：{contact}

感谢您的理解与支持。

{company_name}
{date}
```

## 五、紧急维护

### 5.1 紧急维护条件

| 条件 | 说明 | 优先级 |
|------|------|--------|
| 安全事件 | 安全事件需紧急处理 | P0 |
| 严重故障 | 严重故障需紧急修复 | P0 |
| 数据风险 | 数据风险需紧急处理 | P0 |
| 服务中断 | 服务中断需紧急恢复 | P1 |
| 性能严重下降 | 性能严重下降需处理 | P1 |
| 合规要求 | 合规强制要求 | P1 |

### 5.2 紧急维护流程

```
│ 2. 快速审批                          │
│    - 技术负责人快速审批              │

│    - 电话/即时通讯审批               │
```

| 租户维护 | 7 天 | 3 天 | 租户协商通知 |
| 紧急维护 | 立即 | 立即 | 紧急立即通知 |
| 滚动维护 | 3 天 | 1 天 | 滚动提前通知 |
| 升级维护 | 14 天 | 7 天 | 升级提前通知 |

### 6.2 通知内容

```
【维护通知】
通知编号: {notice_id}
```

## 七、维护期间能力限制

### 7.1 能力限制

| 限制项 | 说明 | 影响程度 |
|--------|------|----------|
| 服务降级 | 部分服务降级 | 中度 |
| 功能限制 | 部分功能限制 | 轻度 |
| 访问限制 | 访问限制 | 重度 |
| 写入限制 | 写入限制 | 重度 |
| 查询限制 | 查询限制 | 中度 |
| 流量限制 | 流量限制 | 中度 |

### 7.2 限制级别

| 级别 | 说明 | 限制内容 | 恢复时间 |
|------|------|----------|----------|
| L1-轻度 | 轻度限制 | 部分功能限制 | < 1小时 |
| L2-中度 | 中度限制 | 服务降级 | < 2小时 |
| L3-重度 | 重度限制 | 服务暂停 | < 4小时 |
| L4-完全 | 完全限制 | 服务完全停止 | > 4小时 |

### 7.3 限制通知

```
【维护期间能力限制通知】
限制级别: {limit_level}
限制内容: {limit_content}
影响功能: {affected_features}
恢复时间: {recovery_time}
替代方案: {alternative}
注意事项: {notes}
```

### 7.4 限制监控

| 监控项 | 说明 | 告警阈值 |
|--------|------|----------|
| 限制生效 | 限制是否生效 | 未生效告警 |
| 限制范围 | 限制范围是否正确 | 范围偏差告警 |
| 限制时长 | 限制时长是否超预期 | 超时告警 |
| 恢复状态 | 恢复是否正常 | 恢复失败告警 |
