# AUTO_SCALING.md - 自动扩缩容

## 目的
定义自动扩缩容的触发条件、策略和执行流程。

## 适用范围
所有需要弹性伸缩的组件，包括服务实例、资源池、缓存容量。

## 扩缩容维度

| 维度 | 指标 | 扩容阈值 | 缩容阈值 | 冷却时间 |
|------|------|----------|----------|----------|
| CPU | 使用率 | >80% | <30% | 5分钟 |
| 内存 | 使用率 | >85% | <40% | 5分钟 |
| QPS | 请求数 | >限值 | <30%限值 | 3分钟 |
| 队列 | 积压数 | >1000 | <100 | 2分钟 |
| 延迟 | 响应时间 | >SLA | <50%SLA | 3分钟 |

## 扩缩容策略

### 水平扩缩容
```yaml
horizontal_scaling:
  min_instances: 2
  max_instances: 20
  scale_up:
    trigger: cpu > 80%
    action: add 2 instances
    cooldown: 300s
  scale_down:
    trigger: cpu < 30% for 10m
    action: remove 1 instance
    cooldown: 600s
```

### 垂直扩缩容
```yaml
vertical_scaling:
  resources:
    cpu:
      min: 1
      max: 8
      step: 1
    memory:
      min: 2GB
      max: 16GB
      step: 2GB
  trigger: memory > 85%
  cooldown: 600s
```

### 预测性扩缩容
```yaml
predictive_scaling:
  enabled: true
  model: linear_regression
  features:
    - historical_qps
    - time_of_day
    - day_of_week
  lookahead: 30m
  buffer: 20%
```

## 执行流程

### 扩容流程
```
指标触发
    ↓
┌─────────────────────────────────────┐
│ 1. 评估扩容需求                      │
│    - 计算所需容量                    │
│    - 检查扩容限制                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行扩容                          │
│    - 创建新实例                      │
│    - 等待就绪                        │
│    - 加入负载均衡                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 验证扩容效果                      │
│    - 检查指标改善                    │
│    - 记录扩容事件                    │
└─────────────────────────────────────┘
```

### 缩容流程
```
指标触发
    ↓
┌─────────────────────────────────────┐
│ 1. 选择缩容目标                      │
│    - 选择负载最低实例                │
│    - 检查最小实例数                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 优雅下线                          │
│    - 标记为下线中                    │
│    - 等待请求完成                    │
│    - 移出负载均衡                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 释放资源                          │
│    - 销毁实例                        │
│    - 记录缩容事件                    │
└─────────────────────────────────────┘
```

## 安全限制

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 最小实例数 | 2 | 保证高可用 |
| 最大实例数 | 100 | 成本控制 |
| 单次扩容上限 | 10 | 防止过激 |
| 扩容冷却时间 | 300s | 防止抖动 |
| 缩容冷却时间 | 600s | 防止误缩 |

## 监控告警

| 事件 | 告警级别 | 通知方式 |
|------|----------|----------|
| 扩容触发 | P2 | IM |
| 扩容失败 | P1 | 短信+IM |
| 达到上限 | P1 | 短信+IM |
| 缩容触发 | P3 | 日志 |

## 维护方式

- 调整阈值: 更新策略配置
- 调整限制: 更新安全限制表
