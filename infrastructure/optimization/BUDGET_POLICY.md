# BUDGET_POLICY.md - 预算策略

## 目的
定义每类任务的预算策略，确保系统知道什么任务值得多花资源。

## 适用范围
所有任务执行的预算控制。

## 预算维度

### Token预算
| 维度 | 说明 |
|------|------|
| input_budget | 输入Token预算 |
| output_budget | 输出Token预算 |
| total_budget | 总Token预算 |

### 工具预算
| 维度 | 说明 |
|------|------|
| max_calls | 最大调用次数 |
| max_retries | 最大重试次数 |
| timeout_ms | 超时时间 |

### 步骤预算
| 维度 | 说明 |
|------|------|
| max_steps | 最大步骤数 |
| max_depth | 最大深度 |
| max_parallel | 最大并行数 |

### 重试预算
| 维度 | 说明 |
|------|------|
| max_retries | 最大重试次数 |
| retry_delay | 重试延迟 |
| backoff_factor | 退避因子 |

### 时间预算
| 维度 | 说明 |
|------|------|
| soft_timeout | 软超时 |
| hard_timeout | 硬超时 |
| progress_interval | 进度报告间隔 |

## 任务预算档位

### 低价值任务
| 任务类型 | Token预算 | 步骤预算 | 时间预算 |
|----------|-----------|----------|----------|
| 简单问候 | 500 | 1 | 5s |
| 常识问答 | 1000 | 2 | 10s |
| 格式转换 | 2000 | 3 | 15s |

**特点**:
- 快速响应优先
- 最小资源消耗
- 不允许重试

### 标准任务
| 任务类型 | Token预算 | 步骤预算 | 时间预算 |
|----------|-----------|----------|----------|
| 一般问答 | 3000 | 3 | 30s |
| 信息搜索 | 5000 | 5 | 30s |
| 文档编辑 | 8000 | 10 | 60s |

**特点**:
- 平衡质量与成本
- 允许有限重试
- 标准超时设置

### 高价值任务
| 任务类型 | Token预算 | 步骤预算 | 时间预算 |
|----------|-----------|----------|----------|
| 复杂分析 | 15000 | 15 | 120s |
| 方案规划 | 20000 | 20 | 180s |
| 文档生成 | 25000 | 25 | 180s |

**特点**:
- 质量优先
- 允许多次重试
- 较长超时时间

### 高风险任务
| 任务类型 | Token预算 | 步骤预算 | 时间预算 |
|----------|-----------|----------|----------|
| 安全操作 | 5000 | 5 | 60s |
| 删除操作 | 3000 | 3 | 30s |
| 配置修改 | 5000 | 5 | 60s |

**特点**:
- 严格步骤限制
- 强制确认流程
- 详细审计记录

## 预算耗尽降级策略

### Token耗尽
| 阶段 | 处理 |
|------|------|
| 80% | 警告，开始简化 |
| 90% | 强制简化输出 |
| 100% | 终止，返回部分结果 |

### 步骤耗尽
| 阶段 | 处理 |
|------|------|
| 80% | 警告，加速执行 |
| 90% | 跳过非关键步骤 |
| 100% | 终止，返回当前结果 |

### 时间耗尽
| 阶段 | 处理 |
|------|------|
| 软超时 | 加速，简化输出 |
| 硬超时 | 终止，返回部分结果 |

## 预算配置格式

```json
{
  "task_type": "search",
  "value_level": "standard",
  "budgets": {
    "token": {
      "input": 3000,
      "output": 2000,
      "total": 5000
    },
    "tool": {
      "max_calls": 5,
      "max_retries": 2,
      "timeout_ms": 30000
    },
    "step": {
      "max_steps": 5,
      "max_depth": 2,
      "max_parallel": 2
    },
    "retry": {
      "max_retries": 2,
      "retry_delay": 1000,
      "backoff_factor": 2
    },
    "time": {
      "soft_timeout": 25000,
      "hard_timeout": 30000,
      "progress_interval": 5000
    }
  },
  "degradation": {
    "on_token_80": "warn",
    "on_token_90": "simplify",
    "on_token_100": "terminate_partial",
    "on_step_80": "warn",
    "on_step_90": "skip_non_critical",
    "on_step_100": "terminate_current"
  }
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 预算配置缺失 | 使用默认预算 |
| 预算值无效 | 使用安全最小值 |
| 降级失败 | 终止并报告 |

## 维护方式
- 新增预算维度: 添加到预算维度表
- 新增任务档位: 添加到任务档位表
- 调整降级策略: 更新降级策略表
