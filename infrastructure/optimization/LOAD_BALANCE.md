# LOAD_BALANCE.md - 负载均衡策略

## 目的
定义负载均衡算法、健康检查和故障转移机制。

## 适用范围
所有需要负载均衡的场景，包括服务调用、任务分发、数据分片。

## 负载均衡算法

| 算法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 轮询 | 服务能力相近 | 简单均匀 | 不考虑实际负载 |
| 加权轮询 | 异构服务 | 灵活 | 权重配置复杂 |
| 最少连接 | 长连接场景 | 动态均衡 | 需要连接计数 |
| 一致性哈希 | 缓存场景 | 减少迁移 | 节点变化影响大 |
| 随机 | 无状态服务 | 简单 | 分布不均 |

```
    ↓

│ 2. 应用负载均衡算法                  │

└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 发起请求                          │
│    - 记录请求开始                    │
│    - 设置超时                        │
└─────────────────────────────────────┘
```

### 权重计算
```javascript
// 动态权重计算
function calculateWeight(node) {
  const baseWeight = node.configWeight || 1;
  const cpuFactor = 1 - node.cpuUsage / 100;
  const memoryFactor = 1 - node.memoryUsage / 100;
  const latencyFactor = node.avgLatency < 100 ? 1 : 0.5;
  
  return baseWeight * cpuFactor * memoryFactor * latencyFactor;
}
```

## 健康检查

### 检查方式
| 类型 | 检查方法 | 频率 | 超时 | 失败阈值 |
|------|----------|------|------|----------|
| TCP | 端口探测 | 10s | 3s | 3次 |
| HTTP | GET /health | 10s | 5s | 3次 |
| 自定义 | 执行脚本 | 30s | 10s | 2次 |

### 健康状态
| 状态 | 说明 | 处理 |
|------|------|------|
| 健康 | 检查通过 | 正常流量 |
| 亚健康 | 偶尔失败 | 降低权重 |
| 不健康 | 连续失败 | 移除流量 |
| 恢复中 | 曾不健康 | 逐步恢复 |

## 故障转移

### 转移策略
| 策略 | 适用场景 | 说明 |
|------|----------|------|
| 重试 | 临时故障 | 重试其他节点 |
| 降级 | 服务不可用 | 返回默认值 |
| 熔断 | 连续失败 | 快速失败 |

### 重试配置
```yaml
retry:
  max_attempts: 3
  backoff:
    initial: 100ms
    max: 1s
    multiplier: 2
  retry_on:
    - timeout
    - 503
```

### 熔断配置
```yaml
circuit_breaker:
  enabled: true
  failure_threshold: 5
  success_threshold: 2
  timeout: 30s
  half_open_requests: 3
```
