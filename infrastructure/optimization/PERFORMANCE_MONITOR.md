# PERFORMANCE_MONITOR.md - 性能监控规范

## 目的
定义性能监控指标、采集频率和告警阈值。

## 适用范围
所有系统组件的性能监控。

## 监控指标

### 系统级指标
| 指标 | 采集频率 | 告警阈值 | 处理动作 |
|------|----------|----------|----------|
| CPU使用率 | 10s | >80% 持续1分钟 | 扩容/限流 |
| 内存使用率 | 10s | >85% | 清理缓存 |
| 磁盘使用率 | 60s | >90% | 清理日志 |
| 网络延迟 | 10s | >500ms | 切换节点 |

### 应用级指标
| 指标 | 采集频率 | 告警阈值 | 处理动作 |
|------|----------|----------|----------|
| 响应时间 | 每请求 | >3s | 降级处理 |
| 错误率 | 10s | >5% | 熔断 |
| QPS | 10s | 超限 | 限流 |
| 并发数 | 10s | 超限 | 排队 |

### 业务级指标
| 指标 | 采集频率 | 告警阈值 | 处理动作 |
|------|----------|----------|----------|
| 任务成功率 | 60s | <95% | 告警 |
| 平均处理时间 | 60s | 超基准20% | 优化 |
| 队列积压 | 10s | >1000 | 扩容 |

## 采集方式

### Agent 采集
```yaml
agent_metrics:
  cpu: /proc/stat
  memory: /proc/meminfo
  disk: df -h
  network: /proc/net/dev
```

### 应用埋点
```javascript
// 请求埋点
metrics.record('request', {
  endpoint: '/api/chat',
  duration: 1234,
  status: 200
});
```

### 日志解析
```yaml
log_metrics:
  error_rate:
    pattern: "ERROR"
    window: 10s
    threshold: 5%
```

## 告警规则

### 告警级别
| 级别 | 触发条件 | 通知方式 | 响应时间 |
|------|----------|----------|----------|
| P0 | 系统不可用 | 电话+短信 | 5分钟 |
| P1 | 核心功能受损 | 短信+IM | 15分钟 |
| P2 | 性能下降 | IM | 1小时 |
| P3 | 异常趋势 | 邮件 | 24小时 |

### 告警聚合
```yaml
aggregation:
  window: 5m
  group_by: [alert_type, severity]
  dedupe: true
  max_alerts: 10
```

## 监控面板

### 实时面板
- 系统资源使用率
- 请求QPS/延迟
- 错误率趋势
- 队列状态

### 历史面板
- 日/周/月趋势
- 同比环比分析
- 容量规划数据

## 维护方式
- 新增指标: 更新指标表
- 调整阈值: 更新告警规则
- 新增面板: 更新面板配置
