# RESOURCE_POOL.md - 资源池管理

## 目的
定义资源池的创建、分配、回收和监控机制。

## 适用范围
所有可复用资源，包括连接池、线程池、内存池。

## 资源池类型

| 类型 | 资源 | 初始大小 | 最大大小 | 用途 |
|------|------|----------|----------|------|
| 连接池 | DB连接 | 10 | 100 | 数据库访问 |
| 线程池 | 执行线程 | 5 | 50 | 并发任务 |
| 内存池 | 内存块 | 64MB | 1GB | 大对象分配 |
| HTTP池 | HTTP连接 | 20 | 200 | 网络请求 |

## 资源分配

### 分配策略
| 策略 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 轮询 | 均衡负载 | 简单 | 不考虑状态 |
| 最少使用 | 长连接 | 均衡 | 需要统计 |
| 加权 | 异构资源 | 灵活 | 配置复杂 |

### 分配流程
```
请求资源
    ↓
┌─────────────────────────────────────┐
│ 1. 检查池状态                        │
│    - 有空闲资源 → 直接分配            │
│    - 无空闲资源 → 检查是否可扩容      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 扩容或等待                        │
│    - 可扩容 → 创建新资源              │
│    - 不可扩容 → 进入等待队列          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 分配资源                          │
│    - 标记为使用中                    │
│    - 记录分配时间                    │
└─────────────────────────────────────┘
```

## 资源回收

### 回收条件
| 条件 | 触发动作 | 说明 |
|------|----------|------|
| 使用完成 | 归还池 | 正常回收 |
| 超时 | 强制回收 | 防止泄漏 |
| 错误 | 销毁重建 | 保证可用 |
| 空闲过长 | 缩容 | 释放资源 |

### 回收流程
```javascript
async function release(resource) {
  // 1. 检查资源状态
  if (resource.isBroken()) {
    await resource.destroy();
    return;
  }
  
  // 2. 重置资源状态
  await resource.reset();
  
  // 3. 归还池
  pool.return(resource);
}
```

## 健康检查

### 检查方式
| 类型 | 检查方法 | 频率 | 超时 |
|------|----------|------|------|
| 连接池 | ping | 30s | 5s |
| 线程池 | 任务探测 | 10s | 1s |
| 内存池 | 内存检测 | 60s | - |

### 异常处理
| 异常 | 处理方式 |
|------|----------|
| 资源泄漏 | 告警 + 强制回收 |
| 池耗尽 | 告警 + 扩容 |
| 资源损坏 | 销毁 + 重建 |

## 监控指标

| 指标 | 计算方式 | 告警阈值 |
|------|----------|----------|
| 使用率 | used/total | >80% |
| 等待数 | wait_queue_size | >10 |
| 平均等待时间 | avg(wait_time) | >1s |
| 泄漏数 | leaked_count | >0 |
