## 平台增强型强内核技能整改启动

### 背景

用户要求将项目从"部署型任务系统"改造成"平台增强型强内核技能"。

### 核心转变

**从**: 部署型任务系统（需要 PostgreSQL/Redis/Docker）
**到**: 平台增强型强内核技能（默认零配置可用）

### 三种运行模式

1. **Skill Default 模式**（默认）
   - SQLite 持久化
   - 单进程
   - request-driven 执行
   - 无外部依赖

2. **Platform Enhanced 模式**
   - 借用平台调度能力
   - 自动降级到 Default

3. **Self-hosted Enhanced 模式**（可选）
   - PostgreSQL/Redis 增强
   - 不影响默认模式

### 已完成工作

1. **skill_entry 层**（入口层）
   - `input_router.py` - 输入路由器
   - `validators.py` - 参数校验器
   - `response_formatter.py` - 响应格式化器
   - `error_codes.py` - 错误码定义

2. **orchestration 层**（编排层）
   - `task_orchestrator.py` - 任务编排器
   - `workflow_orchestrator.py` - 工作流编排器
   - `batch_orchestrator.py` - 批量编排器

### 环境限制

当前沙箱环境：

- 无 sudo 权限
- 无外网访问

因此采用 SQLite + FakeRedis 作为默认实现。
