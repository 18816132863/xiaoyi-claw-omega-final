# 2026-04-25 记忆日志

## V8.8.6 Route Active Execution 完成

### 核心成果

1. **Route 状态机实现**
   - 状态流转: generated → verified → active → deprecated/failed
   - active 状态必须有 smoke_executed + audit_written 证据
   - 文件: `infrastructure/route_state.py`, `infrastructure/route_audit.py`

2. **Connected Runtime 分层验证**
   - 4 种模式: dry_run / fake_device / probe_only / connected_runtime
   - probe_only 禁止所有副作用操作
   - 文件: `config/runtime_modes.py`, `config/runtime_policy.yaml`

3. **设备状态拆分 (5 维度)**
   - session_connected: 小艺 Claw 连接端默认 true
   - runtime_bridge_ready: 运行时桥接状态
   - permission_ready: 权限就绪状态
   - capability_service_ready: 能力服务就绪
   - action_ready: 动作执行就绪
   - 文件: `infrastructure/device_runtime_state.py`

4. **L0 自动降级链**
   - normal_probe → fast_probe → limited_scope_probe → cache_fallback → permission_diagnosis
   - 触发条件: success_rate < 80%
   - 文件: `infrastructure/connected_runtime_recovery_policy.py`

5. **任务超时机制**
   - 20s: heartbeat interval
   - 60s: no progress → degrade
   - 120s: no progress → stop_probe
   - 180s: total → output_partial
   - 文件: `infrastructure/task_progress_guard.py`

6. **故障注入测试**
   - 支持注入 CONTACT/CALENDAR/NOTE/LOCATION 服务超时
   - 验证恢复链正确触发
   - 文件: `scripts/run_fault_injection_tests.py`
