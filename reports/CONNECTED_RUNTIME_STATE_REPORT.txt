[Executive Summary]
本次修复解决了核心逻辑错误：不再因为某个能力 timeout 就判定"设备不可达"。
在小艺 Claw 连接端运行时，默认 session_connected = true，不再误报 "no real device"。

[Key Changes]

1. 设备状态拆分 (device_runtime_state.py)
   - session_connected: 小艺会话是否连接
   - runtime_bridge_ready: call_device_tool / 小艺工具桥是否可用
   - permission_ready: 联系人/日历/备忘录/位置/截图等权限是否可用
   - capability_service_ready: 单个能力服务是否响应
   - action_ready: 是否允许真实动作执行

2. 报告口径修正
   旧口径（错误）:
     - device unavailable
     - no real device
   
   新口径（正确）:
     - session_connected = true
     - connected_runtime = partial
     - 部分能力 timeout 或 permission_required

3. Timeout 分类
   不再笼统判定"设备未连接"，必须区分：
   - contact_service_timeout
   - calendar_service_timeout
   - note_service_timeout
   - permission_required
   - runtime_bridge_unavailable
   - session_disconnected
   
   只有 session heartbeat 失败 + runtime bridge 不存在 + 所有 probe 失败，
   才能判定 session_disconnected。

4. 自动恢复策略 (connected_runtime_recovery_policy.py)
   遇到联系人/日历/备忘录/位置 timeout，自动执行：
   1. retry - 重试
   2. limited_scope_probe - 有限范围探测
   3. cache_fallback - 缓存降级
   4. pending_queue - 待处理队列
   5. permission_diagnosis - 权限诊断
   6. human_action_required - 人工干预（最后手段）

5. L0 成功率自动调整
   如果 L0 success rate < 80%，自动切换：
   normal_probe → fast_probe → limited_scope_probe → cache_fallback → permission_diagnosis

6. 任务进度超时控制
   - 每 20 秒输出当前阶段
   - 超过 60 秒自动降级
   - 超过 120 秒停止当前 probe
   - 超过 180 秒输出部分结果，不继续等待
