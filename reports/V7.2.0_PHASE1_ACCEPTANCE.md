## Workflow 执行方式

通过注册 handler 真实执行：
- `prepare_context` → `prepare_handler`
- `check_architecture_integrity` → `check_handler`
- `generate_report` → `report_handler`

WorkflowEngine `_execute_action()` 规则：
1. 有 action handler → 走 handler
2. 没 handler，但有 `skill_router` → 走 `skill_router.execute()`
3. 两者都没有 → 直接失败（只允许 test/noop/mock 测试动作）
