# 家庭基础信息查询场景

> **场景说明**：当用户说要查询家庭基础信息，统计家庭数、房间数、设备总数、设备在线/离线状态，以家庭账户名称维度进行播报。

## 涉及子模块

**说明**：本场景技能负责家庭基础信息查询的编排，需要调用 common-skill 中的 get_homes_info、get_devices_info、get_device_detail 等基础技能获取数据。
* **get_homes_info (家庭信息查询)**
    * **路由路径：** `common-skill/get_homes_info/SKILL.md`
    * **功能:** 获取家庭列表信息。要输出家庭账户名称
* **get_devices_info (设备基础信息查询)**
    * **路由路径：** `common-skill/get_devices_info/SKILL.md`
    * **功能:** 获取所有设备基础信息列表及状态。
* **get_device_detail (获取设备详细信息)**
    * **路由路径：** `common-skill/get_device_detail/SKILL.md`
    * **功能:** 获取指定设备的详细信息。

## 展示格式

- 总体概览：家庭总数、设备总数
- 家庭详情：
 - 家庭名称，房间总数，设备总数量
  - 房间名称，设备总数量
   - 设备类型，设备名称，设备数量
    - 设备在线状态（必须项，不要丢失）
- 总结说明，并提醒用户检测离线设备

**注意：**
- 子系统不是一个设备，若查询的设备包含子系统，需单独罗列子系统，即设备和子系统需分开说明，数量统计时禁止把子系统归类为设备。

## 展示说明

- 以家庭维度扩展，有几个家庭就罗列几个家庭
- 汇总为表格展示，UX 显示清晰有美感
- 保留关键信息，表格结构精炼化，回复不要啰嗦，用户可以快速直观看到汇总结果
- 不要输出推理过程，直接给出结果

## 强制要求

**查询家庭基础信息时，必须同时查询设备在线/离线状态，这是强制要求，不可省略！**

## 执行流程

1. 调用 get_homes_info 获取家庭列表
2. 调用 get_devices_info 获取所有设备的 deviceId 列表
3. **【强制】** 对每个 deviceId 调用 get_device_detail 获取设备的 status 字段（online/offline）
4. 汇总统计在线设备数量和离线设备数量

## 注意事项

- get_devices_info 返回的数据不包含设备在线状态，必须通过 get_device_detail 逐个查询设备
- 如果某个设备查询识别，标记为"查询失败"并继续查询其他设备
- 子系统类设备（如照明子系统、冷暖新风子系统）可能查询失败，这是正常的
