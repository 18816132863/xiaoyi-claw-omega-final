# 门锁出门/回家留言设置功能Skill

## 1. 必读项

本技能用于**控制**门锁的出门/回家留言设置，包括添加、删除、修改留言等操作。**不支持查询操作，查询操作请参考lock-info**。

### 服务ID（sid）
- **sid**: `homeGreeting`

### 重要说明
- 回家留言和出门留言是不同的功能，执行时注意根据用户要求区分
- 留言的音量大小可以在音量设置-留言/天气提醒音量中进行设置
- 修改留言后，会在下次【门锁在线】时生效，以下任一情况会让【门锁在线】：
  1. 开关门
  2. 产生逗留录像或图片
  3. 有人按门铃
  4. 开启实时视频

### VID说明
- 通过id值区分回家和出门
- 回家取值：1，2，3，4，5，当前回家留言最多5条
- 出门取值：51，52，53，出门留言最多3条
- 配置时已经使用过的id不能再次使用

## 2. 触发关键字

当用户提到以下关键词时，应加载本技能：
- 出门留言
- 回家留言
- 留言设置
- 添加留言
- 删除留言

### 4.2 核心参数说明

| 参数 | 说明 |
|------|------|
| `ts` | 时间戳，配置时需要生成，格式：20191226T075719Z |
| `vid` | 留言id，回家：1-5，出门：51-53 |
| `act` | 操作类型，0-增加配置，1-删除配置，2-修改配置 |
| `uid` | 用户id，"all"表示全部用户，""表示离家语音 |
| `ve` | 留言内容 |
| `type` | 类型，1-文字留言，2-录音留言 |
| `tb` | 音色，0-小艺女声，1-小艺少女，2-保留，3-小艺男声，4-小艺童声 |

## 5. 调用示例

### 5.1 添加回家留言

> **场景**：用户 ZTT 希望添加一条回家提醒："宝贝，回家后记得写作业"
> **前提**：当前已有 1 条留言 (`vid=1`)，上限 5 条，`vid=2` 可用。

##### 配置逻辑与步骤

| 步骤 | 逻辑说明 | 参数映射 |
| :--- | :--- | :--- |
| **1. VID 校验** | 检查 `vid` 是否冲突。回家留言范围 `1~5`。若 `vid=2` 未被占用，则填入。 | `vid`: `2` |
| **2. UID 映射** | 将用户名称 "ZTT" 转换为系统 ID。 | `uid`: `"3"` |
| **3. 时间默认** | 未指定日期时，默认全天生效。 | `st`: `"07:00"`, `et`: `"24:00"`, `rd`: `"1,2,3,4,5,6,7"` |
| **4. 内容润色** | 将指令转化为温暖口语。 | `ve`: `"宝贝，回家后记得写作业"` |
| **5. 属性设定** | 类型设为文字，开关默认开启。 | `type`: `1`, `switch`: `1` |

##### 核心规范 (Critical Rules)
1. **增量更新原则**：`data` 中的 `cfg` 数组**仅包含**本次新增的留言配置，**严禁**携带已存在的其他留言数据。
2. **VID 唯一性**：系统会自动校验 `vid` 是否已存在。若 `1~5` 已满，需提示用户先删除再添加。
3. **修改前必须获得用户授权**：门锁最多存在5条开门留言，3条回家留言，新增留言时，先检查留言是否全部用完，未用完时新增一条，用完时指导用户删除其中一条后再新增留言。
4. **UID 格式**：
   - 指定用户：`"3"`
   - 全部用户：`"all"`
   - 部分用户：`"1,4,5"`
   - 离家语音：`""` (空字符串)
5. **操作选项**：当前为添加，所以"act":0

##### 执行命令 (Add Command)

```bash
node common-skill/bin/smarthome-claw.js control_device \
  --dev-id "xxx" \
  --prod-id "xxx" \
  --operation "POST" \
  --sid "homeGreeting" \
  --data '{"cfg":[{"vid":2,"fid":"","uid":"3","st":"07:00","rd":"1,2,3,4,5,6,7","act":0,"rpt":1,"type":1,"tb":0,"switch":1,"et":"24:00","ve":"宝贝，回家后记得写作业"}],"ts":"20260408T150744Z842"}' \
  --verbose
```

### 5.2 删除回家留言

> **场景**：用户要求删除 财运旺，万事顺 的留言。

##### 配置逻辑与步骤

1. **查询定位**：系统根据用户意图，先查询当前已存在的留言列表，定位目标 `vid` (本例为 `2`)。
2. **构建删除包**：构造仅包含目标留言配置的 JSON 对象。
3. **发送指令**：通过 `POST` 请求触发删除逻辑。

##### 核心规范 (Critical Rules)
- **精准匹配**：`cfg` 数组中**必须且只能**包含待删除的那一条留言配置。
- **忽略冗余**：即使删除命令中包含了其他字段（如时间、内容），系统也仅以 `vid` 作为唯一标识进行删除操作，**无需**传入其他已存在留言的完整配置。

##### 执行命令 (Delete Command)

```bash
node common-skill/bin/smarthome-claw.js control_device \
  --dev-id "xxx" \
  --prod-id "xxx" \
  --operation "POST" \
  --sid "homeGreeting" \
  --data '{"cfg":[{"vid":2,"fid":"","uid":"all","st":"07:00","rd":"1,2,3,4,5,6,7","act":0,"rpt":1,"type":1,"tb":0,"switch":1,"et":"24:00","ve":"财运旺，万事顺"}],"ts":"20260408T150744Z842"}' \
  --verbose
```
*(注：实际调用时，`ve` 内容可保持任意值，系统主要依据 `vid` 识别删除目标)*

## 6. 注意事项

- 本技能仅支持控制，不支持查询操作
- **在执行任何删除类操作时，均需要提醒用户进行授权，禁止未经用户同意私自执行删除操作**
- 门锁最多存在5条回家留言，3条出门留言
