# 门锁历史记录查询功能Skill

## 1. 必读项

本技能用于**查询**门锁的历史记录，包括开关门记录、回家记录、开门记录、门铃记录、逗留抓拍、门铃响铃、告警记录等。**不支持控制操作**。

### 记录类型说明

通过`eventData`事件中的字段进行区分：
- `type`: 区分告警和正常门锁事件，0-记录，1-告警
- `up`: 区分用户操作类型
  - 0-指纹开锁
  - 1-密码开锁
  - 2-人脸开锁
  - 3-门卡开锁
  - 4-手表手环开锁
  - 5-华为钱包开锁
  - 6-一次性密码开门
  - 7-物理钥匙开门
  - 8-添加密码
  - 9-删除密码
  - 10-添加指纹
  - 11-删除指纹
  - 12-添加人脸
  - 13-删除人脸
  - 14-添加门卡
  - 15-删除门卡
  - 16-添加手表手环
  - 17-删除手表手环
  - 18-添加钱包钥匙
  - 19-删除钱包钥匙
  - 20-添加用户
  - 21-删除用户
  - 22-门铃响铃
  - 23-编辑用户
  - 24-出门抓拍
  - 25-反锁
  - 26-解除反锁
  - 27-开启布防
  - 28-解除布防
  - 29-敲门事件
  - 30-连续按门铃
  - 31-门内sensor检测靠近
  - 32-开始视频通话
  - 33-开始视频录制
  - 34-双重认证开锁成功
  - 35-室内一握开锁
  - 36-室内旋钮开锁
  - 37-钥匙或旋钮开锁
  - 38-上锁成功
  - 39-周期密码开门
  - 40-添加密码
  - 41-删除密码
  - 42-胁迫指纹开锁
  - 43-关门待上锁
  - 44-手机远程开锁
  - 45-中控屏远程开锁
  - 47-掌静脉开锁
  - 48-添加掌静脉
  - 49-删除掌静脉
  - 50-门铃呼手机记录
  - 51-删除无感开锁钥匙
  - 52-无感开锁
  - 53-添加无感开锁钥匙
  - 54-删除无感开锁钥匙
  - 55-视频留言

- `das`: 区分告警类型
  - 1-门未关告警（门锁管理）
  - 2-门虚掩告警（门锁管理）
  - 3-门未锁告警(故障)（门锁管理）
  - 4-低电告警（低功耗/设备管理）
  - 5-网络质量差（电子猫眼）
  - 6-指纹连续错误
  - 7-密码连续错误
  - 8-人脸连续错误
  - 10-挟持告警事件（门锁管理）
  - 12-布防告警（门锁管理）
  - 13-可疑逗留（设备管理）
  - 14-被撬告警（设备管理）
  - 15-遮挡告警（电子猫眼）
  - 16-过曝告警（电子猫眼）
  - 17-长时间未开门（智能家居云）
  - 18-离家抓拍
  - 19-应急供电
  - 20-NFC开门连续错误
  - 21-NFC禁用告警
  - 22-前面板测距传感器脏污告警
  - 23-后面板测距传感器脏污告警
  - 24-"小红健"功能介绍
  - 25-锂电池低电
  - 26-干电池低电
  - 27-掌静脉连续错误
  - 28-超声脏污提醒
  - 29-长时间门未关告警

- `rt`: 记录类型，0-普通，1-视频，2-照片
- `et`: 事件发生时间，格式如20191226T075719Z

## 4. 相关参数说明

### 4.1 服务ID（sid）

| sid | 功能      | 说明          |
|-----|---------|-------------|
| `eventData` | 门锁事件    | 查询门锁所有记录，包括告警和正常事件 |

### 4.2 eventData参数说明

| 参数 | 说明 |
|------|------|
| `data` | data字段为下列字段组成的json格式的字符串 |
| `type` | 固定枚举范围,区分告警和正常门锁事件,0-记录,1-告警 |
| `uic` | 取值范围0-100,用户头像ID |
| `un` | 用户名称 |
| `up` | 区分用户操作，详见上方操作类型说明 |
| `das` | 区分告警类型，详见上方告警类型说明 |
| `rt` | 记录类型,0-普通,1-视频2,-照片 |
| `cl` | 绑定联系人使用emergencyContact服务中的联系人id,多个id用,隔开 |
| `aid` | 1、上报好望云报警事件的id，用于获取视频流;2、up=50-门铃呼叫手机记录时内容："接听结果:通话时长" |
| `et` | 事件发生时间,如20191226T075719Z |

## 5. 调用示例

### 5.1 查询某个用户的历史记录

##### 默认近 24 小时
```bash
node common-skill/bin/smarthome-claw.js get_device_histories \
  --dev-id "xxx" \
  --sid "eventData" \
  --verbose
```

##### 今天
```bash
node common-skill/bin/smarthome-claw.js get_device_histories \
  --dev-id "xxx" \
  --sid "eventData" \
  --date today
  --verbose
```

##### 昨天
```bash
node common-skill/bin/smarthome-claw.js get_device_histories \
  --dev-id "xxx" \
  --sid "eventData" \
  --date yesterday
  --verbose
```

##### 最近 7 天
```bash
node common-skill/bin/smarthome-claw.js get_device_histories \
  --dev-id "xxx" \
  --sid "eventData" \
  --last-days 7
  --verbose
```

##### 最近 30 天
```bash
node common-skill/bin/smarthome-claw.js get_device_histories \
  --dev-id "xxx" \
  --sid "eventData" \
  --last-days 30
  --verbose
```

### 5.3 数据展示格式

开头固定：当前您家门锁记录如下：
- 展示开关门记录（按时间倒序）
- 展示门铃响铃记录（如有）
- 展示逗留抓拍记录（如有）
- 展示告警记录（如有）

## 6. 注意事项

- 本技能仅支持查询，不支持控制操作
- 历史记录查询使用 `get_device_histories` 命令
- 记录类型通过 `up` 和 `das` 字段进行区分
